# PLAN.114 — Router State Machine (`routerReducer`)

## Context

Plan 113 (spike) validated the TCA design. This plan implements the full router reducer,
state types, and environment interface in `packages/router-browser`. It is the TCA
counterpart to `BrowserRouter` (plan 95): `BrowserRouter` is the imperative foundation;
`routerReducer` + `routerStore` is the declarative state-machine layer on top.

The store is a singleton — navigation state is global. It uses valtio's proxy so that
both React components (via `useSnapshot`) and non-React code (via `.getProxyState()`)
can subscribe.

## State, Actions, Environment

```typescript
// packages/router-browser/src/router-store/types.ts

export type RouterState<Route> =
  | { tag: 'idle';       route: Result<Route, string> }
  | { tag: 'navigating'; from: Result<Route, string>; to: Route }
  | { tag: 'committed';  route: Route }
  | { tag: 'error';      route: Route; error: unknown };

export type RouterAction<Route> =
  | { tag: 'navigate';          route: Route }
  | { tag: 'replace';           route: Route }
  | { tag: 'popstate';          url: string }
  | { tag: 'loader-resolved';   route: Route }
  | { tag: 'loader-failed';     route: Route; error: unknown }
  | { tag: 'prefetch';          route: Route };

export interface RouterEnv<Route> {
  parseUrl:      (url: string) => Result<Route, string>;
  printRoute:    (route: Route) => string;
  pushState:     (url: string) => Effect<never>;
  replaceState:  (url: string) => Effect<never>;
  subscribePopstate: Effect<RouterAction<Route>>; // streaming — never terminates
  runLoaders:    (route: Route) => Effect<RouterAction<Route>>;
}
```

## Reducer Logic

```
navigate(route):
  - $.state = { tag: 'navigating', from: current, to: route }
  - return Effect.merge(env.pushState(url), env.runLoaders(route))

replace(route):
  - $.state = { tag: 'navigating', from: current, to: route }
  - return Effect.merge(env.replaceState(url), env.runLoaders(route))

popstate(url):
  - parse url → route
  - if parse fails: $.state = { tag: 'error', ... }
  - else: $.state = { tag: 'navigating', from: current, to: route }
  - return env.runLoaders(route)

loader-resolved(route):
  - $.state = { tag: 'committed', route }

loader-failed(route, error):
  - $.state = { tag: 'error', route, error }

prefetch(route):
  - return env.runLoaders(route)  [result discarded — warms loader cache only]
```

## Success Criteria

- All five action types covered by tests with `mockRouterEnv`.
- `navigate` → `loader-resolved` cycle transitions through `navigating` → `committed`.
- `popstate` parses the URL via `env.parseUrl` (not `router.parse` directly — testable).
- `loader-failed` transitions to `error` state without throwing.
- `withLogging` works out of the box (reducer uses tagged actions).
- No `any` in the public types.
