# PLAN.123 — `renderRoute()` Server Render Function

## Context

With `StaticRouter` (plan 120), `dehydrate` (plan 121), and head management (plan 122)
in place, the server render function is the composition point: it takes a `Request`,
runs loaders, renders the React tree to a streaming HTML response, embeds the dehydrated
loader data, and emits the correct `<head>` tags.

The BFF uses Hono with `renderToReadableStream` (WinterCG), so the render function
targets that API directly.

## Goal

```typescript
// packages/router-react/src/server.ts  (subpath: @arbor/router-react/server)

export interface RenderOptions<Route> {
  loaderMap?: LoaderMap<Route>;
  metaMap?:   MetaMap<Route>;
  bootstrapScripts?: string[];  // JS bundle paths to include as <script>
  onError?: (error: unknown) => void;
}

export async function renderRoute<Route extends { tag: string }, Map>(
  request: Request,
  router: RouterContract<Route, Map>,
  App: React.ComponentType,
  options?: RenderOptions<Route>,
): Promise<Response>
```

### Implementation

1. Parse `request.url` → `Result<Route, string>`. If parse fails, return a plain 404
   response (no React render required).
2. Create `StaticRouter(router, url)`.
3. Run loaders (if `loaderMap` provided) for the matched route's ancestor chain — same
   parallel execution as plan 101's client-side path.
4. Create a `StaticProvider` context that carries the `StaticRouter` + loader data.
5. Call `renderToReadableStream(<StaticProvider><App /></StaticProvider>, { bootstrapScripts, onError })`.
6. `collectHead()` after the stream starts (React renders synchronously for the shell).
7. Return a `Response` with:
   - `Content-Type: text/html`
   - HTML body: doctype + `<head>` (with collected tags) + streamed body
   - Embedded `<script>window.__ARBOR_DATA__ = ...</script>` from `dehydrate(route, data)`

### Package structure

`packages/router-react` gains a new subpath export:

```json
"./server": {
  "import": "./dist/server.js",
  "types": "./dist/server.d.ts"
}
```

This keeps server-only code (React DOM server imports) out of the browser bundle. The
client-side barrel (`@arbor/router-react`) never imports from `server.ts`.

## Error handling

- Loader error: return a `Response` with status 500 and a plain HTML error page. Do not
  attempt to render the React tree with failed data.
- React render error: `onError` callback is called; the stream completes with partial HTML.
  The client re-renders on hydration — graceful degradation.
- Parse failure (unknown URL): 404 response, no render.

## Success Criteria

- `renderRoute` returns a `Response` with `Content-Type: text/html`.
- The HTML contains the correct route-matched content (not a blank page).
- `window.__ARBOR_DATA__` script tag is present in the HTML.
- `<title>` and `<meta>` tags are in `<head>`.
- A 404 is returned for an unrecognized URL.
- The `./server` subpath is not included in the browser bundle (tree-shake test).
