# Plan 55 — Rename `_ctx`/`Ctx` → `_meta`/`Meta` on RouteNode

## Context

Plan 54 added `Ctx` as the 5th type parameter on `RouteNode` and typed the `_ctx`
field per factory. This left `RouteNode` with two confusingly similar names sharing
the same interface:

- `Context` (3rd param) — the DI bag injected into route handlers.
- `Ctx` (5th param) — the plugin metadata bag (HTTP method, schemas, etc.).

`Ctx` is an abbreviation of the same word as `Context`, making the distinction
opaque. `_meta` / `Meta` is the correct term: it names _data about the route_
(method, schemas, rate-limit policy) rather than data _carried into handlers_.
The name is also idiomatic — Vue Router and TanStack Router both call per-route
configuration data `meta`.

Note: `_ctxMap` (on the router object) maps route tags to their DI `Context`
types and is therefore correctly named — it is **not** renamed here.

## Goal

- `_ctx` field → `_meta` on `RouteNode` and all sites.
- `Ctx` type parameter → `Meta` on `RouteNode` and all sites.
- `getHttpCtx` → `getHttpMeta`, `getOpenApiCtx` → `getOpenApiMeta`.
- Verification chain green, no test files deleted.
