# PLAN.115 — React TCA Bindings (`RouterStoreProvider`)

## Context

Plan 114 ships `createRouterStore(router)` — a singleton valtio-backed store. This plan
wires it into React, providing an alternative to the hooks-based `RouterProvider` (plan 97)
for teams that use the `Store` pattern.

The two providers are co-equal exports — teams choose one. Both expose the same hooks
(`useRoute`, `useNavigate`, `useMatch`), so component code is identical regardless of
which provider is used at the root. The TCA provider adds:
- `useRouterStore()` — direct access to `[state, send]` for advanced use cases.
- `withLogging` opt-in via a `debug` prop on the provider.
- The router store composes with other application stores (e.g., authStore) via the same
  `useSnapshot` / valtio subscription model the rest of the app uses.

## API Surface

```typescript
// packages/router-react/src/router-store-provider.tsx

<RouterStoreProvider store={routerStore} debug={import.meta.env.DEV}>
  {children}
</RouterStoreProvider>

// All hooks from plan 97 remain identical:
const route = useRoute<Route>();        // Result<Route, string>
const navigate = useNavigate<Route>();  // (route: Route) => void
const match = useMatch('user');         // { tag: 'user', id: string } | null

// TCA-only hook:
const [state, send] = useRouterStore<Route, RouterAction<Route>>();
```

## Implementation Notes

`RouterStoreProvider` differs from plan 97's `RouterProvider` in one key way: it does NOT
create its own subscription to the popstate event. Instead, it fires
`send({ tag: 'popstate', url: window.location.href })` on mount to initialize the route,
and the `subscribePopstate` streaming effect in `RouterEnv` handles all subsequent events.
This means the popstate subscription is managed entirely by the effect system, not React.

The `debug` prop wraps the reducer with `withLogging` from `@arbor/common`. In production
builds, `debug={false}` (default) passes the raw reducer with no overhead.

Singleton vs. per-tree: the store is created outside React (like `authStore`). The
provider is just a React context carrier for the store reference. Multiple trees can share
the same store — navigation state is truly global.

## Dependencies

- Plan 114 (`routerReducer`, `createRouterStore`).
- Plan 97 (hooks layer — `useRoute`, `useNavigate`, `useMatch` must be shared).
- `packages/common` `Store`, `useStore`, `withLogging`.

## Success Criteria

- `useRoute()`, `useNavigate()`, `useMatch()` work identically under both providers.
- `withLogging` logs actions to console when `debug={true}`.
- Mounting `RouterStoreProvider` starts the `subscribePopstate` streaming effect.
- Unmounting does NOT stop the streaming effect (singleton store).
- `useRouterStore()` returns live valtio snapshot and `send`.
- No `any` in public hooks.
