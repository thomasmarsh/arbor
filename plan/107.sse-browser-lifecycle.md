# PLAN.107 — Browser SSE Lifecycle (`useSSE` hook)

## Context

`createSseClient()` in `@arbor/router` already provides a typed SSE client. In a React
component, the subscription must be tied to the component lifecycle: open on mount, close
on unmount, reconnect on disconnect. This plan wraps the existing client with React hooks.

## Goal

1. `useSSE(sseRoute, opts)` hook:
   - Opens the SSE connection on mount.
   - Returns `{ events: E[], status: 'connecting' | 'open' | 'closed' | 'error' }` where
     `E` is the typed event type inferred from the `sseRoute` definition.
   - Closes the connection and clears listeners on unmount.
   - Reconnects with exponential backoff (base: 1 s, max: 30 s, jitter: ±20%) on
     unexpected closure.
   - Accepts `maxEvents?: number` (default: 100) to bound memory for long-lived streams.

2. `useSseCallback(sseRoute, handler)` — fire-and-forget variant: calls `handler(event)`
   for each incoming event; does not accumulate events in state.

## API Surface

```typescript
const { events, status } = useSSE(routes.feed, { baseUrl: 'https://api.example.com' });
// events: FeedEvent[]
// status: 'connecting' | 'open' | 'closed' | 'error'

useSseCallback(routes.notifications, (event) => {
  toast(event.message); // event is fully typed
});
```

## Success Criteria

- Connection opens on mount; closes on unmount.
- Events accumulate in `events` array; typed correctly.
- Reconnect fires after unexpected close; does not reconnect on intentional unmount.
- `maxEvents` evicts oldest events when limit is reached.
- No `EventSource` memory leaks after unmount.
