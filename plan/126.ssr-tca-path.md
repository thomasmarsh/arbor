# PLAN.126 — SSR + TCA Path (`staticRouterEnv`)

## Context

The TCA router store (plans 113–117) uses `liveRouterEnv` which touches `window.history`
and `window.addEventListener`. On the server, these do not exist. This plan adds a
`staticRouterEnv` for use during SSR and ensures `RouterStoreProvider` (plan 115) can
be initialized from server-dehydrated state.

## Goal

1. `staticRouterEnv<Route>(router, url, loaderMap?)` — a server-safe `RouterEnv`:

   ```typescript
   export function staticRouterEnv<Route>(
     router: RouterContract<Route, any>,
     url: string,
     loaderMap?: LoaderMap<Route>,
   ): RouterEnv<Route> {
     return {
       parseUrl:          (u) => router.parse(u),
       printRoute:        (r) => router.print(r),
       pushState:         () => Effect.none(),   // no-op
       replaceState:      () => Effect.none(),   // no-op
       subscribePopstate: Effect.none(),         // no-op streaming
       runLoaders:        (r) => loaderMap
                                 ? Effect.tryPromise(...)
                                 : Effect.send({ tag: 'loader-resolved', route: r }),
       updateMeta:        () => Effect.none(),   // no-op on server (head collected separately)
       saveScroll:        () => Effect.none(),
       restoreScroll:     () => Effect.none(),
     };
   }
   ```

2. `createStaticRouterStore<Route>(router, url, loaderMap?)` — creates a `Store` from
   `staticRouterEnv`. The initial state is `committed` (no `navigating` phase — the server
   knows the URL upfront and loaders are awaited before rendering).

3. `RouterStoreProvider` (plan 115) must handle initial state injection:

   ```typescript
   <RouterStoreProvider store={staticRouterStore} dehydratedData={serverData}>
     <App />
   </RouterStoreProvider>
   ```

   On the client, when `dehydratedData` is provided, the router store is initialized in
   `committed` state with the server-provided route + loader data, skipping the
   `navigating` phase entirely.

4. `dehydrateStore(store)` — serializes `RouterState<Route>` (specifically the `route`
   and `data` fields from the `committed` state) as JSON. Mirrors plan 121's `dehydrate()`
   but operates on the TCA store's state rather than a raw loader-data map.

## Server Render Flow (TCA Path)

```
1. BFF receives Request
2. staticRouterStore = createStaticRouterStore(router, url, loaderMap)
3. store.send({ tag: 'navigate', route: parsedRoute }) → runs loaders via Effect
4. Await committed state (poll or await Effect completion)
5. storeData = dehydrateStore(staticRouterStore)
6. renderToReadableStream(
     <RouterStoreProvider store={staticRouterStore}>
       <App />
     </RouterStoreProvider>
   )
7. Embed storeData in <script>window.__ARBOR_STORE__ = ...</script>
```

## Client Bootstrap (TCA Path)

```typescript
const storeData = rehydrateStore(); // reads window.__ARBOR_STORE__
const routerStore = createRouterStore(router, loaderMap, storeData);
//                                                       ^ initialized from server state

hydrateRoot(root, <RouterStoreProvider store={routerStore}><App /></RouterStoreProvider>);
```

## Await Committed State

Step 4 above requires waiting for the effect-driven loader to complete before rendering.
The `Store` does not currently expose a "wait for condition" API. Two approaches:

- **Promise-based**: `staticRouterStore.waitFor(s => s.tag === 'committed', timeoutMs)`
  — new method on `Store` that returns a Promise resolving when the predicate is true.
- **Effect-based**: the loader effect completes synchronously (since `staticRouterEnv`
  uses `Effect.tryPromise` which forks a fiber) — but fiber completion is async. Must
  await the Promise from the loader.

Plan 119's spike should validate which approach is cleaner.

## Success Criteria

- `staticRouterEnv` has no `window.*` references.
- `createStaticRouterStore` initializes in `committed` state after loaders complete.
- `dehydrateStore` → `rehydrateStore` round-trip produces correct initial client state.
- Client `RouterStoreProvider` with dehydrated data skips the `navigating` phase.
