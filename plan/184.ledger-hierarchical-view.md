# 184. Ledger Hierarchical View (Epic → Story → Task)

## Context

The current table always groups by execution status (In Progress / Ready / Blocked / Done). There is no way to navigate by project hierarchy. The `ledger.jsonl` already has `epic` and `story` metadata rows; the API's `parseLedger` currently discards them.

## Goal

Add a "tree" view mode: Epic → Story → Tasks, collapsible at both levels. Flat status-grouped view remains the default. A toolbar toggle switches between them.

## Change Surface & T-shirt Sizing

**M**

### API (`apps/api/src/ledger/`)

| File | Change |
|---|---|
| `reader.ts` | Parse `epic` and `story` rows in `parseLedger`; return `{ tasks, waves, epics, stories }` |
| `routes.ts` | New `GET /api/ledger/hierarchy` → `{ epics: EpicEntry[], stories: StoryEntry[], tasks: TaskEntry[] }` (flat arrays — client builds the tree) |

`EpicEntry`: `{ id, title }`. `StoryEntry`: `{ id, epic, title, layer }`. Both shapes already exist in the JSONL; just need Zod schemas and API surface.

### Store (`apps/ui/src/ledger/`)

| File | Change |
|---|---|
| `ledger.store.ts` | Add `viewMode: 'flat' \| 'tree'` to state; `epicMeta: EpicEntry[]`; `storyMeta: StoryEntry[]`; `collapsedEpics: Set<string>`; `collapsedStories: Set<string>`; add `toggleViewMode`, `toggleEpicCollapse`, `toggleStoryCollapse`, `fetchHierarchy` actions |
| `ledger.env.ts` | Add `fetchHierarchy` method to env |

### UI (`apps/ui/src/ledger/`)

| File | Change |
|---|---|
| `LedgerTable.tsx` | Render tree mode: `EpicGroupRow` → `StoryGroupRow` → `TaskRow`; arrow nav skips collapsed groups |
| `LedgerToolbar.tsx` | View mode toggle button (flat / tree) |
| New `EpicGroupRow.tsx` | MUI `TableRow` with chevron, epic title, task count badge |
| New `StoryGroupRow.tsx` | Indented MUI `TableRow` with chevron, story title |

In tree mode, filters still apply to tasks; collapsed groups with zero matching tasks are hidden.

## Dependencies

181 (selection model must be stable — `selectedId` not `selectedIndex` — before tree navigation is added)

## Success Criteria

- Tree view shows all tasks organized under their epic and story.
- Clicking a chevron (or pressing `→`/`←` on a group row) collapses or expands the group.
- Collapsed state survives filter changes and list reloads.
- Arrow navigation moves through visible rows only (skips collapsed subtrees).
- Flat view is completely unaffected by this plan.
- New API endpoint has a contract test.
