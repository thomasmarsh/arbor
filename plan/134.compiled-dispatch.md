# PLAN.134 — Compiled Dispatch: Map-Indexed Route Lookup

## Context

Plan 132 spike validated that `walkParse()` performs a linear scan across all sibling
nodes at every level of the route tree. For a tree of 500 routes, a match takes ~1666 ns
and a no-match takes ~2073 ns, both growing linearly with tree size.

The spike's `compile-dispatch.ts` prototype demonstrated a compiled Map-of-literals tree:
at each dispatch level, literal segments are bucketed into a `Map<string, SubTree>` for
O(1) lookup. Non-literal nodes (`:param`, `#param`, optional, wildcard) fall back to a
short ordered list. Correctness was verified against `walkParse` for all segment kinds.

Measured speedups at startup cost ~101 µs for 500 routes (one-time, negligible):

| Routes | `walkParse` match | compiled dispatch | match speedup | no-match speedup |
|-------:|------------------:|------------------:|--------------:|-----------------:|
|     20 |            596 ns |            198 ns |         3.0×  |             15×  |
|    100 |            809 ns |            183 ns |         4.4×  |             69×  |
|    500 |           1666 ns |            183 ns |          9×   |            218×  |

Plan 74 (radix tree) is canceled. This plan supersedes it for the common case.

## Goal

Replace the hot path in `defineRoutes().parse()` with an indexed dispatch that:

1. Pre-builds an `IndexedTree` once at `defineRoutes()` call time.
2. Uses `Map<string, IndexedWalkNode[]>` at every level for literal-segment dispatch.
3. Falls back to a short ordered list for non-literal segments.
4. Preserves all existing behavior: nested `{ child }` structure, query param extraction,
   schema validation, `ParseDiag` collection.

`walkParse` (the exported function) stays unchanged — it is used directly by tests and
internal recursion, and changing its signature would be unnecessary churn.

## Change Surface

### `packages/router/src/core/walk.ts`

Add:

```typescript
type IndexedWalkNode = WalkNode & {
  _index: {
    literals: Map<string, IndexedWalkNode[]>;
    nonLiterals: IndexedWalkNode[];
  };
};

export function indexNodes(nodes: WalkNode[]): IndexedWalkNode[];

export function walkParseIndexed(
  nodes: IndexedWalkNode[],
  urlSegments: string[],
  query: URLSearchParams,
  params?: Record<string, unknown>,
  diag?: ParseDiag[],
): Record<string, unknown> | null;
```

`indexNodes` is a pure startup function: recursively wraps each `WalkNode` with a
`_index` field containing the literal bucket map and non-literal list for that node's
children. Cost: O(N) over all nodes. Idempotent.

`walkParseIndexed` mirrors `walkParse` exactly but uses `node._index` instead of
iterating `node.children` linearly. Internal recursion also uses `walkParseIndexed`.

### `packages/router/src/core/define-routes.ts`

At the start of `defineRoutes()`:

```typescript
const indexedNodes = indexNodes(nodes);
```

Replace both `walkParse(nodes, ...)` call sites (lines 129 and 142) with
`walkParseIndexed(indexedNodes, ...)`.

### Tests

Add to `walk.test.ts`:

- Mirror a representative subset of existing `walkParse` tests against `walkParseIndexed`
  to ensure the indexed path produces identical output.
- One test verifying `indexNodes` builds a map with the correct literal buckets.

No new example files needed — the external behavior is unchanged.

## Algorithm

`indexNodes(nodes)` builds `_index` for each node's children, not its own segments.
This mirrors the recursion structure of `walkParse`: a node's `_index` is used when
dispatching into its children.

```
indexNodes([n1, n2, n3]):
  for each node ni:
    recurse: indexNodes(ni.children)    // bottom-up
    build ni._index from ni.children:
      literals = Map { first-lit-seg → [children whose first seg is that literal] }
      nonLiterals = [children with non-literal or absent first segment]
  return nodes with _index attached
```

`walkParseIndexed(indexedNodes, urlSegments, ...)`:

```
1. first = urlSegments[0]
2. if first is defined:
     bucket = node._index.literals.get(first)   // could be undefined
     for each indexed node in bucket (typically 1):
       result = try match
       if result → return
3. for each node in node._index.nonLiterals:
       result = try match
       if result → return
4. return null
```

Within each iteration, the per-node logic (`matchSegments`, `handleLeafNode`,
`validateSchema`, child recursion) is identical to `walkParse`. Only the iteration
strategy differs.
