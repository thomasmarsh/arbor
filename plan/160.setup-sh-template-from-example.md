# Plan 160 ‚Äî Rewrite `arbor setup` to Template from `.env.example`

## Context

`scripts/setup.sh` was the original source of this plan: it embedded env file content as
heredocs and drifted from the canonical `.env.example` files. That script has since been
deleted and its logic ported to `tools/arbor-cli/src/arbor_cli/cmd_setup.py` (`arbor setup`).

The port was a faithful translation ‚Äî the heredocs moved from bash into Python string
literals, so the same maintenance divergence remains:

```python
# cmd_setup.py ‚Äî still duplicates content that lives in apps/*/env.example
_create_env(
    Path("apps/bff/.env.local"),
    "ARBOR_OIDC_ISSUER=http://localhost:8080/realms/arbor\n"
    "ARBOR_OIDC_CLIENT_ID=arbor-bff\n"
    ...
)
```

The `.env.example` files in `apps/api/` and `apps/bff/` are the canonical templates.
`arbor setup` should derive from them rather than duplicate them.

The one thing `cmd_setup.py` must still do that `.env.example` cannot: generate a
cryptographically random `ARBOR_SESSION_SECRET` and splice it in. That logic is preserved
(`secrets.token_hex(32)` already replaces the old `node crypto` call).

## Goal

Rewrite `cmd_setup.py::setup()` to:

1. Copy `apps/api/.env.example` ‚Üí `apps/api/.env.local` (skip if exists).
2. Copy `apps/bff/.env.example` ‚Üí `apps/bff/.env.local`, then replace the
   `ARBOR_SESSION_SECRET=` line with a freshly-generated 64-char hex secret.
3. Copy `apps/bff/.env.example` ‚Üí `apps/bff/.env.staging.local` with the same secret
   substitution (skip if exists).

Adding a new env var to an app now requires only editing that app's `.env.example`.
`cmd_setup.py` requires no change.

## Change Surface & T-shirt Sizing

**XS** ‚Äî one function rewritten in one file.

| File                                          | Change                                                                                                                                                       |
|-----------------------------------------------|--------------------------------------------------------------------------------------------------------------------------------------------------------------|
| `tools/arbor-cli/src/arbor_cli/cmd_setup.py`  | Replace hardcoded string literals in `setup()` with `Path.read_text()` copies + `re.sub()` secret injection; delete the now-redundant `_create_env` helper  |

### Implementation sketch

```python
import re, secrets
from pathlib import Path

def setup():
    """Scaffold .env.local files for each app (skips existing files)."""
    secret = secrets.token_hex(32)

    _copy_env(Path("apps/api/.env.example"),      Path("apps/api/.env.local"))
    _copy_env(Path("apps/bff/.env.example"),      Path("apps/bff/.env.local"),      secret)
    _copy_env(Path("apps/bff/.env.example"),      Path("apps/bff/.env.staging.local"), secret)
    ...

def _copy_env(src: Path, dst: Path, session_secret: str | None = None) -> None:
    if dst.exists():
        typer.echo(f"skipped  {dst} (already exists)")
        return
    content = src.read_text()
    if session_secret:
        content = re.sub(
            r"^ARBOR_SESSION_SECRET=.*$",
            f"ARBOR_SESSION_SECRET={session_secret}",
            content,
            flags=re.MULTILINE,
        )
    dst.write_text(content)
    typer.echo(f"created  {dst}")
```

## Dependencies

## Success Criteria

- Running `arbor setup` on a clean checkout produces `apps/api/.env.local`,
  `apps/bff/.env.local`, and `apps/bff/.env.staging.local`.
- Each file matches the corresponding `.env.example`, with `ARBOR_SESSION_SECRET`
  replaced by a 64-char hex string.
- Adding a variable to `apps/bff/.env.example` is immediately reflected the next time
  `arbor setup` runs on a clean checkout ‚Äî no `cmd_setup.py` edits required.
- `diff <(grep -v ARBOR_SESSION_SECRET apps/bff/.env.local) <(grep -v ARBOR_SESSION_SECRET apps/bff/.env.example)`
  produces no output.
