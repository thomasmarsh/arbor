# PLAN.104 — Code Splitting with `lazyRoute`

## Context

Large applications need route-level code splitting: the JS bundle for `/admin` should not
be loaded until the user navigates to `/admin`. The arbor route tree is defined at startup
and is fully typed — code splitting must not break type inference.

The key constraint: the route _type_ (its tag, params, response shapes) must be statically
available at compile time even if the _component_ is lazily loaded. This is achievable
because the route schema is in `@arbor/router`; only the React component is split.

## Goal

1. `lazyRoute(tag, () => import('./pages/UserPage'))` — associates a lazy component import
   with a route tag. The component is loaded only when the route is first navigated to.
2. On navigation to a lazy route: the module load and the data loader fire in parallel
   (`Promise.all`). Neither waits for the other.
3. `<RouteSwitcher>` component — renders the correct (possibly lazy) component for the
   current route. Wraps each lazy route in a `React.Suspense` boundary automatically.
4. Type safety: `lazyRoute` infers the tag from the route union and requires the imported
   module to export a component that accepts `useLoaderData(tag)` types.

## API Surface

```typescript
const lazyMap = createLazyMap(router, {
  user:  () => import('./pages/UserPage'),
  users: () => import('./pages/UserListPage'),
});

// In JSX:
<RouteSwitcher lazyMap={lazyMap} fallback={<PageSkeleton />} />
```

## Change Surface

- New: `packages/router-react/src/lazy-route.ts` (`createLazyMap`)
- New: `packages/router-react/src/route-switcher.tsx` (`RouteSwitcher`)
- New tests: lazy component loads only on first navigation to that tag

## Dependencies

- Plan 101 (data loaders — parallel load + fetch).
- Plan 102 (navigation lifecycle — pending state during module load).

## Success Criteria

- Component module is NOT imported until the first navigation to its tag.
- Module load and loader fire in parallel.
- `<RouteSwitcher>` renders the correct component with no `any` in the component→data
  prop connection.
- Subsequent navigations to the same tag use the cached module.
