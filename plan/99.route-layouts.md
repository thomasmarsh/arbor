# PLAN.99 — Route Layouts and Nesting

## Context

The arbor route tree is hierarchical: `{ tag: 'user', id: '42', child: { tag: 'user-tab', tab: 'posts' } }`.
A layout should render once for the parent subtree and stay mounted when child routes
change — the same pattern as React Router's `<Outlet>` or Next.js nested layouts.

The discriminated union makes this mechanical: a layout component for `users` is active
when `route.tag === 'users'` OR when the flattened route contains any route whose parent
chain passes through `users`. The route tree structure encodes this.

## Goal

1. `<RouteLayout tag="users">` — renders `children` when the current route is in the
   `users` subtree; stays mounted while navigating between child routes.
2. `useChildRoute(tag)` — returns the child route for the given parent tag, narrowed to
   the correct child union type.
3. Route tree traversal helper: `isDescendantOf(route, tag)` — checks if a route object
   is a descendant of a given tag without walking the tree manually.

## API Surface

```typescript
// Layout wrapper — mounts when route is in subtree, stays mounted for child navigations
<RouteLayout tag="users" fallback={<NotFound />}>
  <UserListPage />  // receives useChildRoute('users') automatically
</RouteLayout>

// Hook: access the child route within a layout
const child = useChildRoute('users'); // narrowed to child union of 'users'
```

## Implementation Notes

The `FlattenChildrenImpl` type in `@arbor/router` already produces the full flat union.
`isDescendantOf` needs to walk the `child` property of the discriminated union recursively,
which is runtime logic mirroring the type-level flatten. This is straightforward: every
route node in the discriminated union has an optional `child` property.

`RouteLayout` must NOT re-render children when the route tag matches the same parent but
a different child — use a `useMemo` or `React.memo` boundary on the layout itself.

## Success Criteria

- `RouteLayout` renders children when the route is anywhere in the subtree.
- Parent layout does NOT unmount when navigating between sibling child routes.
- `useChildRoute` is typed to the correct child union.
- `fallback` renders when no route matches the subtree.
