# PLAN.110 — `@arbor/router-devtools`

## Context

Debugging browser navigation is painful without visibility into the current route, loader
states, and real-time connection health. A DevTools panel or overlay that reads from the
same `BrowserRouter` instance gives instant insight into route parsing, loader timing, and
SSE/WS connection state. This is the same pattern as React Query DevTools and Redux DevTools.

## Goal

`packages/router-devtools` — a React component that renders a collapsible overlay panel:

1. **Route inspector**: current route tag + all extracted params; full route object as
   formatted JSON; parse error message if the URL failed to parse.
2. **Navigation history**: the last N routes visited; click to re-navigate (dev only).
3. **Loader status**: per-route loader state (pending / resolved / error); timing in ms;
   cached vs. fresh fetch indicator.
4. **Real-time connections**: list of active SSE subscriptions and WebSocket connections
   with their current status and event/message count.
5. **Route tree**: expandable tree of all registered routes with their tags and segment
   patterns.

## API Surface

```typescript
import { RouterDevtools } from '@arbor/router-devtools';

// Render anywhere in the tree; reads from RouterProvider context
<RouterDevtools position="bottom-right" defaultOpen={false} />
```

## Packaging Decisions

- Renders as a floating overlay, not a browser extension (simpler distribution; no Chrome
  Web Store review cycle).
- Production bundle guard: `RouterDevtools` is a no-op (renders `null`) when
  `process.env.NODE_ENV === 'production'`.
- Zero runtime cost in production via tree-shaking (`if (import.meta.env.DEV)` guard on
  the subscription to `BrowserRouter`).

## Change Surface

- New: `packages/router-devtools/` (new workspace package)
- New: panel components, router context subscription, CSS-in-JS (or CSS modules)

## Dependencies

- Plans 97–109 (React bindings, loaders, SSE/WS hooks).
- Must not change any existing package APIs — reads state via context only.

## Success Criteria

- Panel renders in dev; no-ops in prod.
- Current route updates live on navigation.
- Loader states update in real time during navigation.
- No impact on prod bundle size (verified by size check in CI).
