# 186. Ledger Inline Task Editing

## Context

After plan 181 removes letter-key mutation shortcuts, there is no fast path for editing a task. The detail drawer (plan 179) handles full edits but requires two interactions to open. Inline cell editing provides a fast path for common changes: task text, status, and size.

## Goal

Click any editable cell (or press `e`/`F2` with a row selected) to enter edit mode for that cell. Supported inline fields: `text`, `status`, `size`. Other fields (wave, story, epic, deps) remain drawer-only. `Escape` cancels. `Enter` commits with an optimistic update.

## Dependencies

181 (nav overhaul — `selectedId` model required to know which task to enter edit mode on); existing mutation pattern from plans 145/146 to follow for optimistic update structure.

## Success Criteria

- Clicking the text cell of a selected row renders an inline `<TextField>`; `Enter` saves, `Escape` cancels; the row returns to read mode.
- `e` or `F2` on a selected row enters text edit mode without a mouse click.
- Status cell click renders an inline `<Select>` with all valid statuses.
- Optimistic update: the row reflects the change immediately; re-fetch occurs on success.
- `Escape` cancels an in-progress edit before it would close the drawer.
- API endpoint has a contract test; store edit lifecycle has unit tests.
