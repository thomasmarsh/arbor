# Plan 170 — Scalar OpenAPI Demo in apps/api/

## Context

`@arbor/router` natively generates OpenAPI 3.1 specs via `generateSpec()`. The `apps/api/` demo app has real HTTP routes (`usersRouter`, `ledgerRouter`, `helloRouter`) defined with `httpRoute()`, but nothing surfaces the spec or provides a UI to explore it. Scalar is the current best-of-breed OpenAPI UI — lightweight, beautiful, and trivially embeddable via a single HTML file or CDN script tag.

## Goal

Add two new endpoints to `apps/api/src/index.ts`:

1. `GET /openapi.json` — serves the generated OpenAPI 3.1 spec for all routes (`apiRouter` + `ledgerRouter`)
2. `GET /scalar` — serves Scalar UI pointing at `/openapi.json`

No new packages, no build step. Scalar is served via its CDN script tag (`@scalar/api-reference`). The spec endpoint is powered entirely by the existing `generateSpec()` call.

## Change Surface & T-shirt Sizing

**Size: XS** — single file change (`apps/api/src/index.ts`), no new files needed beyond the two Hono route handlers.

Files touched:
- `apps/api/src/index.ts` — add two Hono route handlers after the existing `app.all('/api/*', ...)` block

## Implementation

### 1. Generate the spec

Combine both routers into one spec at startup (static, generated once):

```typescript
import { generateSpec } from '@arbor/router';

const spec = generateSpec(defineRoutes([
  ...apiRouter,      // helloRouter.children + usersRouter.children
  ...ledgerRouter,   // ledgerRouter children
]), { title: 'Arbor API', version: '0.0.0' });
```

`apiRouter` is already a flat array of `RouteNode`s from `defineRoutes(...)`. `ledgerRouter` is the same shape. Pass both as children to a combined `defineRoutes([...apiRouterChildren, ...ledgerRouterChildren])`.

### 2. Serve `/openapi.json`

```typescript
app.get('/openapi.json', (c) => c.json(spec));
```

### 3. Serve `/scalar`

```typescript
app.get('/scalar', (c) =>
  c.html(`<!doctype html>
<html>
  <head><title>Arbor API — Scalar</title><meta charset="utf-8" /></head>
  <body>
    <script
      id="api-reference"
      data-url="/openapi.json"
      src="https://cdn.jsdelivr.net/npm/@scalar/api-reference"></script>
  </body>
</html>`),
);
```

No authentication, no config. The CDN script auto-renders when it finds `#api-reference[data-url]`.

## Dependencies

- `generateSpec` — already exported from `@arbor/router` (`framework/router/src/openapi/generate-spec.ts`)
- Scalar CDN — no install, loaded in the browser

## Success Criteria

- `GET /openapi.json` returns a valid OpenAPI 3.1 JSON object listing all `api/users`, `api/ledger/*`, and `hello` routes
- `GET /scalar` renders the Scalar UI in a browser with all routes browsable and tryable
- No new dependencies added to `apps/api/package.json`
- `pnpm --filter @arbor/api typecheck` passes
