# 183. Ledger Column Reordering via DnD

## Context

Column order in `LedgerTable` is hardcoded. This plan uses the DnD framework from plan 182 to let users drag column headers to reorder them, with the preference persisted in `localStorage`.

## Goal

Drag a column header to a new position; the table re-renders with the new column order. Preference persists across browser sessions via `localStorage`.

## Change Surface & T-shirt Sizing

**S**

| File | Change |
|---|---|
| `ledger.store.ts` | Add `columnOrder: readonly string[]` to `LedgerState`; add `reorderColumns` action; initial value from `localStorage` or hardcoded default |
| `LedgerTable.tsx` | Wrap `<TableHead>` in DnD sortable context from plan 182 adapter; derive rendered columns from `state.columnOrder`; column header cells expose drag handle |
| `ledger.env.ts` | No change needed — localStorage write in reducer side-effect |

Column definitions are extracted into a `COLUMNS` constant (id → label + render fn) so the order array drives the render loop.

## Dependencies

182 (DnD framework spike — adapter API must be settled before this plan can be implemented)

## Success Criteria

- Dragging a column header to a new position reorders the table immediately.
- Column order survives a page refresh (persisted in `localStorage`).
- Drag does not interfere with click-to-select on rows (plan 181).
- All existing table tests pass.
