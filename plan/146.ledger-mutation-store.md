# Plan 146 — Ledger Mutation Actions in TCA Store

## Context

The mutation routes from Plan 145 exist on the API. This plan extends the ledger store (Plan 143)
with actions for setting task status and bumping/deferring rank, wired to the new PATCH endpoints
via typed `createClient` calls.

## Goal

The store can dispatch `setStatus`, `bump`, and `defer` actions. The action optimistically updates
local state, calls the API, then updates state with the server response.

## New Actions

```typescript
// ledger.store.ts additions

export type LedgerAction =
  | { tag: 'fetch' }
  | { tag: 'loaded'; groups: DisplayGroupsResponse }
  | { tag: 'error'; message: string }
  // mutations
  | { tag: 'setStatus'; taskId: number; status: TaskStatus }
  | { tag: 'bump'; taskId: number; waveRanks: number[] }
  | { tag: 'defer'; taskId: number; waveRanks: number[] };
```

## Reducer Cases

```typescript
case 'setStatus': {
  // Optimistic: update the task in local groups immediately
  mutateTaskInGroups($.state, action.taskId, (t) => ({ ...t, status: action.status }));
  return env.setStatus(action.taskId, action.status).map(() => ({ tag: 'fetch' }));
}
case 'bump': {
  const newRank = Math.max(1, Math.min(...action.waveRanks) - 10);
  mutateTaskInGroups($.state, action.taskId, (t) => ({ ...t, rank: newRank }));
  return env.setRank(action.taskId, newRank).map(() => ({ tag: 'fetch' }));
}
case 'defer': {
  const newRank = Math.max(...action.waveRanks) + 10;
  mutateTaskInGroups($.state, action.taskId, (t) => ({ ...t, rank: newRank }));
  return env.setRank(action.taskId, newRank).map(() => ({ tag: 'fetch' }));
}
```

After the API call succeeds the store re-fetches to get the authoritative sorted state.

`mutateTaskInGroups` is a small pure helper that finds the task by id across inProgress/ready/blocked
arrays and applies an updater function.

## Env Extensions

```typescript
// ledger.env.ts additions
export interface LedgerEnv {
  fetchQueue: Effect<Result<DisplayGroupsResponse, string>>;
  setStatus: (id: number, status: TaskStatus) => Effect<void>;
  setRank: (id: number, rank: number) => Effect<void>;
}

// liveLedgerEnv additions
setStatus: (id, status) =>
  Effect.fromPromise(async () => {
    await client.fetch({ tag: 'ledger-patch-task-status', id }, { body: { status } });
  }),
setRank: (id, rank) =>
  Effect.fromPromise(async () => {
    await client.fetch({ tag: 'ledger-patch-task-rank', id }, { body: { rank } });
  }),
```

(Move `client` to module scope so it's shared across all env methods.)
