# 188. arbor CLI — Postgres Backend

## Context

After plan 187 creates the ledger schema and seeds it, the CLI's data layer (`ledger.py`) still reads from JSONL. This plan switches reads and writes to Postgres, makes `plan/ledger.jsonl` a derived VCS artifact (exported on demand), and adds two new commands: `arbor snapshot` and `arbor validate`.

## Goal

1. **Postgres backend**: replace `build_hierarchical_ledger` (JSONL parser) and `update_task` (line-rewriter) with psycopg2 queries against the four ledger tables.
2. **`arbor snapshot`**: dump the DB back to `plan/ledger.jsonl` in canonical order (epics → stories → waves → tasks sorted by wave position + rank). This is the VCS commit artifact.
3. **`arbor validate`**: cross-check DB ↔ plan docs:
   - Every `task.file` in DB → `plan/{file}` must exist on disk.
   - Every `plan/NNN.*.md` → a task with `id = NNN` must exist in DB.
   - Exits non-zero if any mismatch found (suitable for CI).

## Change Surface & T-shirt Sizing

**M**

### `tools/arbor-cli/src/arbor_cli/ledger.py`

- Remove `build_hierarchical_ledger`, `update_task`, `find_ledger` (JSONL-specific).
- Add `get_pg_conn()` — reads `ARBOR_PG_URL` from env (same var used by the Node API); returns a `psycopg2` connection.
- Add `load_ledger_from_pg(conn) -> Ledger` — queries all four tables, assembles the same `Ledger` / `EpicNode` / `StoryNode` / `TaskEntry` model so all downstream functions (`compute_queue`, `compute_display_groups`, etc.) are unchanged.
- Add `update_task_pg(conn, task_id, updates: dict)` — `UPDATE ledger_tasks SET ... WHERE id = $1`.

All existing commands (`next`, `set`, `bump`, `defer`, `tui`, `task`, `plan`) get a one-line change: `load_ledger_from_pg(conn)` instead of `build_hierarchical_ledger(...)`.

### `tools/arbor-cli/src/arbor_cli/commands.py`

- `set_cmd`, `bump_cmd`, `defer_cmd`: call `update_task_pg` then `snapshot` automatically (keep JSONL in sync after every write).

### New: `tools/arbor-cli/src/arbor_cli/cmd_snapshot.py`

```python
def snapshot_cmd():
    """Export DB → plan/ledger.jsonl for VCS commit."""
    conn = get_pg_conn()
    ledger = load_ledger_from_pg(conn)
    write_ledger_jsonl(ledger, REPO_ROOT / "plan" / "ledger.jsonl")
    typer.echo("✓ plan/ledger.jsonl updated")
```

`write_ledger_jsonl` emits rows in canonical order: meta (if any), then epics, stories, waves, tasks.

## Dependencies

187 (schema and seed must be applied before the Postgres backend can function)

## Success Criteria

- `arbor next` works identically to before, now reading from Postgres.
- `arbor set 181 next` updates the DB row; `plan/ledger.jsonl` is auto-updated; `git diff` shows the change.
- `arbor snapshot` produces a `plan/ledger.jsonl` that round-trips cleanly (content equivalent to a re-seed).
- `arbor validate` exits 0 on a clean repo; exits 1 with a clear error message when a plan doc is missing or an orphaned doc exists.
- `ARBOR_PG_URL` not set → clear error message pointing to `arbor db up`.
