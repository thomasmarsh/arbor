# Plan 173 — `IxSession<Before, After, A>`: Core Type

## Context

> **North star**: `plan/spec.ws-session-ergonomics.md` — the complete target API
> from protocol definition through server handler to client. Read it first.

Spike 172 evaluated four session encoding strategies for typed WebSocket protocols; Spike 172b
deepened the CPS + callback injection approach. The verdict (172-session-ergonomics.ts Verdict
section, 172b-cps-deep.ts Synthesis table) is:

> **`IxSession<Before, After, A>`** is the ergonomic runtime form of `Channel<S>`.
> It is the indexed monad over the session state — `.then()` sequences state
> transitions; the compiler rejects wrong-order chains.

This plan adds the core type to the codebase as **additive exports** alongside the existing
`Channel<S>` and `Session` primitives in `src/core/session.ts`. No existing code is touched.

Key reference material is in:
- `scratch/172-session-ergonomics.ts` — Part 5, `IxSession` sketch
- `scratch/172b-cps-deep.ts` — Part 2 (Done token), Part 3 (ChoiceResult), Part 5 (IxSession)

### What `IxSession` is

```typescript
// A protocol computation that:
//   - starts in session state Before
//   - ends in session state After
//   - yields value A
//
// Implementation: CPS. The runner takes a continuation for A and calls it.
// Before/After are phantom types — never used at runtime.
class IxSession<_Before, _After, A> {
  constructor(private readonly _run: <R>(k: (a: A) => Promise<R>) => Promise<R>) {}

  then<NewAfter, B>(
    f: (a: A) => IxSession<_After, NewAfter, B>
  ): IxSession<_Before, NewAfter, B> {
    return new IxSession((k) => this._run((a) => f(a)._run(k)))
  }

  run(): Promise<A> {
    return this._run((a) => Promise.resolve(a))
  }
}
```

### Done token

The `Done` token (unique symbol) is the unforgeable proof of protocol completion.
`IxSession<S, End, Done>` is the canonical "complete session" type for both parties.
See 172b-cps-deep.ts Part 2 for the full rationale.

```typescript
declare const _done: unique symbol
export type Done = typeof _done
export const done: Done = _done as Done  // implementation provides this value
```

### ChoiceResult<C>

`Channel<Select<C>>` in the current session.ts is:
```typescript
{ select<K>(k: K): Channel<C[K]> }  // wrong: caller picks k synchronously
```

The correct form: the holder AWAITS the peer's tag, then routes. `ChoiceResult<C>` is a
discriminated union on `tag` so that narrowing `choice.tag === 'leave'` also narrows
`choice.pick()`'s return type.

```typescript
export type ChoiceResult<C extends Record<string, Session>> = {
  [K in keyof C]: { tag: K; pick(): IxSession<C[K], any, any> }
}[keyof C]
```

## Notes

- `core/` must never import from `contexts/`, `server/`, `openapi/`, or `client/`.
- The `_run` field on `IxSession` is private; the class is not designed to be subclassed.
- `done` is the only inhabitant of `Done`. The implementation provides it to callbacks;
  external callers cannot construct `Done` values without `as any`.
- The `_Before` and `_After` phantom parameters have leading underscores per the codebase
  convention for phantom fields (`_type`, `_child`, `_meta`) — though here they're type
  parameters, not fields.
