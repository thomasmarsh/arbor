# Plan 145 — Ledger Mutation Routes (PATCH status & rank)

## Context

Wave 27 delivers a read-only view. This is the first Wave 28 task, adding the API endpoints
for mutating task state. The `writer.ts` module from Plan 140 already has `updateTask` — this
plan wraps it in typed HTTP routes.

## Goal

Two PATCH routes live on the arbor ledger router:
- `PATCH /api/ledger/tasks/:id` — update `status`
- `PATCH /api/ledger/tasks/:id/rank` — update `rank` (bump/defer)

## Change Surface & T-shirt Sizing

**Size: S**

Modified files:
- `packages/api/src/ledger/routes.ts` — add two new route tags + handlers
- `packages/api/src/ledger/schemas.ts` — add body schemas for patch requests

## Route Definitions

```typescript
// New tag schemas
const PatchTaskStatus = z.object({
  tag: z.literal('ledger-patch-task-status'),
  id: z.coerce.number(),
});
const PatchTaskStatusBody = z.object({ status: TaskStatus });

const PatchTaskRank = z.object({
  tag: z.literal('ledger-patch-task-rank'),
  id: z.coerce.number(),
});
const PatchTaskRankBody = z.object({ rank: z.number() });

// Added to defineRoutes([...])
httpRoute(PatchTaskStatus, 'PATCH', 'api/ledger/tasks/:id', {
  body: PatchTaskStatusBody,
  response: { 200: TaskEntry, 404: ErrorResponse },
}),
httpRoute(PatchTaskRank, 'PATCH', 'api/ledger/tasks/:id/rank', {
  body: PatchTaskRankBody,
  response: { 200: TaskEntry, 404: ErrorResponse },
}),
```

## Handlers

```typescript
'ledger-patch-task-status': async (ctx) => {
  updateTask(ledgerPath(), ctx.params.id, { status: ctx.body.status });
  const { tasks } = parseLedger(ledgerPath());
  const task = tasks.find((t) => t.id === ctx.params.id);
  return task ? respond(200, task) : respond(404, { error: `Task ${ctx.params.id} not found` });
},
'ledger-patch-task-rank': async (ctx) => {
  updateTask(ledgerPath(), ctx.params.id, { rank: ctx.body.rank });
  const { tasks } = parseLedger(ledgerPath());
  const task = tasks.find((t) => t.id === ctx.params.id);
  return task ? respond(200, task) : respond(404, { error: `Task ${ctx.params.id} not found` });
},
```

## Bump/Defer Rank Computation

The rank value for bump/defer is computed on the **client side** (same logic as Python TUI):

```typescript
// Client sends the computed rank value; server just stores it
// bump: max(1, min(waveRanks) - 10)
// defer: max(waveRanks) + 10
```

Computing on the client keeps the API surface simple (just `{ rank: number }`) and avoids
adding wave-awareness to the mutation handler.
