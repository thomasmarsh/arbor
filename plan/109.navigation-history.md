# PLAN.109 — Typed Navigation History

## Context

Browser history is already managed by the History API, but `window.history` only gives
you an opaque length — not a typed record of which routes were visited. `useRouteHistory()`
provides an in-memory typed history stack that survives re-renders and enables application-
level undo/redo for multi-step flows (wizards, forms, deep-drill navigation).

This is separate from the browser's built-in back/forward — it tracks the semantic route
history within the current session, complementing the History API.

## Goal

1. `useRouteHistory()` hook — returns:
   ```typescript
   {
     past: Route[];       // routes visited before current, oldest-first
     current: Route;      // the committed current route
     future: Route[];     // routes after current if user has gone back
     canGoBack: boolean;
     canGoForward: boolean;
     goBack(): void;      // equivalent to navigate to past[last]
     goForward(): void;
   }
   ```
2. History is bounded (default max 50 entries) — oldest entries evicted when full.
3. `navigate(route)` clears the forward stack (same as browser history).

## Success Criteria

- `past` grows correctly with each navigation.
- `goBack()` navigates to the previous route and moves current to `future`.
- `navigate()` from a back-navigated state clears `future`.
- History is bounded; oldest entries drop when limit is exceeded.
- `past` and `future` entries are typed as `Route` (not `string`).
