# 181. Ledger Navigation & Mouse Overhaul

## Context

`useLedgerKeyboard.ts` conflates two concerns: navigation (`j/k/↑/↓`) and ad-hoc mutation triggers (`n=next`, `d=done`, `b=bump`, `D=defer`, `a=all`, `r=refresh`). There is no mouse support — clicking a row does nothing. The store's `selectedIndex: number` is fragile: if the visible list changes (filter, reload), the cursor silently jumps to a different task.

## Goal

Replace the current keyboard/selection model with standard conventions:

- Navigation: arrow keys and vi keys (`hjkl`) only; clicking a row also selects it.
- Remove all letter-key mutation shortcuts. Mutations happen through the drawer (plans 179, 186) or toolbar actions.
- `?` opens a help overlay listing all keybindings.
- `Enter` → open detail drawer. `Escape` → close it.
- Selection is stable across list reloads (tracks by task ID, not array index).

## Change Surface & T-shirt Sizing

**S**

| File | Change |
|---|---|
| `ledger.store.ts` | Replace `selectedIndex: number` with `selectedId: number \| null`; add `selectRow`, recalculate `selectedIndex` as a derived value on the view side from `selectedId` + current row list; remove `bump`/`defer`/`toggleShowAll` actions (or leave bump/defer — but remove their keyboard triggers) |
| `useLedgerKeyboard.ts` | Strip to: `↑/k` → selectUp, `↓/j` → selectDown, `Enter` → openDetail, `Escape` → closeDetail, `?` → openHelp |
| `LedgerTable.tsx` | Add `onClick` handler on `TaskRow` → `send({ tag: 'selectRow', taskId })`; remove hint `Typography` at bottom; render `HelpOverlay` |
| `HelpOverlay.tsx` (new) | MUI `Dialog` listing all active keybindings in a two-column layout |
| `ledger.store.test.ts` | Update tests for `selectedId` model; add click-to-select test |
| `LedgerTable.test.tsx` | Add test: click row → row becomes selected |

## Dependencies

None.

## Success Criteria

- Clicking any visible row selects it (MUI `selected` highlight).
- `j/k/↑/↓` navigate; selection follows the same task after a list reload (e.g., hitting `r` keeps cursor on the same task).
- `?` opens a modal; `Escape` or clicking outside closes it.
- No letter key triggers a mutation directly.
- Full test suite passes with no deleted tests.
