# Plan 191 — Task Status Audit Trail

## Context

`ledger_tasks` has no timestamp columns. When a task transitions between statuses
(todo → next → in_progress → done, or re-opened, or canceled), nothing records when
it happened. The only ordering signal available is task ID, which reflects creation
order — not activity order. It is impossible to answer "what finished recently?" or
"how long did task N sit in-progress?" with any precision.

## Goal

Add a `ledger_task_events` append-only event log that records every status transition
with a `TIMESTAMPTZ` timestamp. Derive convenience columns (`started_at`,
`completed_at`) on `ledger_tasks` from this log via trigger. Expose recent completions
in `arbor task --done` and carry events through the JSONL snapshot.

## Design

### Timestamp convention

`TIMESTAMPTZ` everywhere — Postgres stores as UTC epoch internally regardless of
session timezone. Never use bare `TIMESTAMP` (no zone) for any ledger column. Machine
consumers receive ISO-8601 UTC strings (`2026-06-05T14:32:00Z`); display layers
convert to local time.

### `ledger_task_events` — the source of truth

```sql
CREATE TABLE ledger_task_events (
  id         BIGSERIAL    PRIMARY KEY,
  task_id    INT          NOT NULL REFERENCES ledger_tasks(id),
  from_status TEXT,                         -- NULL for the initial insert event
  to_status   TEXT        NOT NULL,
  changed_at  TIMESTAMPTZ NOT NULL DEFAULT NOW()
);
```

Every status change appends one row. The full history of a task is `SELECT * FROM
ledger_task_events WHERE task_id = N ORDER BY changed_at`.

### Convenience columns on `ledger_tasks`

| Column | Type | Meaning |
|---|---|---|
| `started_at` | `TIMESTAMPTZ` | first transition into `in_progress` |
| `completed_at` | `TIMESTAMPTZ` | most recent transition into `done` or `canceled` |

Both are nullable. They are **derived** — set by an `AFTER INSERT OR UPDATE` trigger
that writes to `ledger_task_events` and then updates `ledger_tasks` in the same
transaction. This means every write path (CLI `arbor set`, API `PATCH`, seed) gets
full audit coverage without caller changes.

### Trigger

```sql
CREATE OR REPLACE FUNCTION record_task_status_change()
RETURNS TRIGGER LANGUAGE plpgsql AS $$
BEGIN
  -- fire on INSERT or on UPDATE where status actually changed
  IF (TG_OP = 'INSERT') OR (OLD.status IS DISTINCT FROM NEW.status) THEN
    INSERT INTO ledger_task_events (task_id, from_status, to_status)
    VALUES (NEW.id,
            CASE WHEN TG_OP = 'UPDATE' THEN OLD.status ELSE NULL END,
            NEW.status);

    IF NEW.status = 'in_progress' AND OLD.started_at IS NULL THEN
      NEW.started_at := NOW();
    END IF;
    IF NEW.status IN ('done', 'canceled') THEN
      NEW.completed_at := NOW();
    END IF;
  END IF;
  RETURN NEW;
END;
$$;

CREATE TRIGGER task_status_audit
BEFORE INSERT OR UPDATE ON ledger_tasks
FOR EACH ROW EXECUTE FUNCTION record_task_status_change();
```

### JSONL snapshot

`TaskEntry` gains:

```python
started_at:   Optional[datetime] = None   # serialised as ISO-8601 UTC "Z" suffix
completed_at: Optional[datetime] = None
events: List[TaskEvent] = Field(default_factory=list)
```

`TaskEvent`:

```python
class TaskEvent(BaseModel):
    from_status: Optional[str]
    to_status:   str
    changed_at:  datetime   # always UTC
```

`write_ledger_jsonl` serialises events inline on each task row. `arbor db seed`
upserts events into `ledger_task_events` — safe to re-run (idempotent on `id` PK).

### CLI surface

- `arbor task --done [-n N]` — tasks ordered by `completed_at DESC` (default N=10),
  showing the timestamp.
- `arbor history N` — full event log for task N with timestamps.
- Normal `arbor task` ready-queue is unchanged.

### API surface

No new routes needed. `GET /api/ledger/tasks` response body gains `started_at`,
`completed_at`, and `events[]` once the query selects them. A future plan can add
`GET /api/ledger/tasks/:id/events` if needed.

## Dependencies

None — standalone DB + CLI change.

## Success Criteria

- Migration applies cleanly (`arbor db migrate`).
- `arbor set N done` appends a row to `ledger_task_events` and sets `completed_at`; verifiable via `psql`.
- `arbor set N todo` after done appends another event row; `completed_at` reflects the most recent close.
- `arbor task --done` lists tasks ordered by `completed_at DESC` with UTC-sourced local timestamps.
- `arbor history N` prints all events for task N in chronological order.
- `arbor db seed` round-trips events from JSONL without duplicating rows.
- No bare `TIMESTAMP` (without zone) columns introduced anywhere.
- `arbor validate` still passes.
