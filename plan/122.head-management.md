# PLAN.122 — Head Management (Server-Safe `<title>` / `<meta>`)

## Context

Plan 111 writes `document.title` on navigation — browser-only, no-op on the server.
SSR requires `<title>` and `<meta>` tags to be present in the server-rendered HTML
`<head>` so crawlers and link-preview scrapers see them. Client-only title updates are
invisible to them.

React 19 introduces native `<title>` and `<meta>` hoisting — rendering `<title>foo</title>`
anywhere in the tree causes React to move it into `<head>` during both SSR and CSR.
React 18 does not do this.

This plan determines the approach and implements it.

## React Version Decision (from plan 119 spike)

_Fill in after plan 119 answers Q7. Placeholder paths:_

**Path A — React 19**: Render `<title>{meta.title}</title>` directly in the router. React
19 hoists it automatically. No extra infrastructure needed. Update plan 111 to use this
instead of `document.title = ...`.

**Path B — React 18**: Implement a `HeadContext` that collects `<title>` / `<meta>` tags
server-side and re-injects them client-side. Pattern: a `HeadProvider` with a mutable
buffer; `useHead(tags)` pushes tags on render; `collectHead()` drains the buffer for
`renderRoute()` to embed in the HTML response.

## Goal (Path B, in case React 18)

1. `HeadProvider` — React context provider with a mutable tag buffer.
2. `useHead({ title, description?, og? })` hook — registers tags for the current render.
   On the client, writes to `document.title` and `<meta>` elements. On the server, pushes
   to the buffer.
3. `collectHead()` — drains the server buffer and returns an HTML string of `<title>` and
   `<meta>` tags for embedding in the response.
4. Integration with plan 111 (route meta map): the `RouterProvider` calls `useHead` with
   the current route's meta on each committed navigation.

## Server-Safe Guard

The key invariant: `useHead` must not call `document.title` during SSR. Guard:

```typescript
if (typeof document !== 'undefined') {
  document.title = title;
}
```

Or better — inject an `updateTitle: (title: string) => void` function from the
`RouterAdapter` (plan 120), where `StaticRouter` provides a no-op and `BrowserRouter`
provides the `document.title` write. This keeps browser globals out of the hook body
entirely.

## Success Criteria

- `collectHead()` returns the correct `<title>` and `<meta>` strings after a server render.
- No `document` access during `renderToReadableStream` (confirmed by running in Node.js

- The React 19 path (if applicable) is simpler: just `<title>` in JSX, no context needed.
