# Plan 179 — LedgerDetailDrawer: Task Details + Plan Document Viewer

## Context

Selecting a task in the table gives no way to see its full details (epic, story, layer, deps,
kind) or read the associated plan document. The `TaskEntry.file` field holds a filename like
`"175.ws-session-route.md"` that lives in the `plan/` directory at the workspace root.

This plan adds:

1. **Two new reducer actions** — `openDetail` / `closeDetail` — following the existing TCA
   pattern. All async work (fetching the plan doc) is modelled as an `Effect` returned from
   the reducer; the env supplies the fetch function.
2. **A new API endpoint** — `GET /api/ledger/tasks/#id/plan` — reads the plan file from disk
   and returns its raw Markdown content.
3. **`LedgerDetailDrawer`** — a MUI `<Drawer anchor="right">` component that is purely
   presentational, reading state from the store and dispatching actions via `send`.

### New reducer state

```typescript
type PlanDocState =
  | { tag: 'idle' }
  | { tag: 'loading'; taskId: number }
  | { tag: 'loaded'; taskId: number; content: string }
  | { tag: 'error'; taskId: number; message: string };

interface LedgerState {
  // ... existing fields unchanged ...
  detailTaskId: number | null;   // which task's drawer is open
  planDoc: PlanDocState;         // load state for that task's plan .md
}
```

Initial values: `detailTaskId: null`, `planDoc: { tag: 'idle' }`.

### New actions

```typescript
| { tag: 'openDetail'; taskId: number }
| { tag: 'closeDetail' }
| { tag: 'planDocLoaded'; taskId: number; content: string }
| { tag: 'planDocError'; taskId: number; message: string }
```

### Reducer logic

```typescript
case 'openDetail': {
  $.state.detailTaskId = action.taskId;
  $.state.planDoc = { tag: 'loading', taskId: action.taskId };
  return env.fetchPlanDoc(action.taskId).map((result) =>
    result.fold<LedgerAction>(
      (content) => ({ tag: 'planDocLoaded', taskId: action.taskId, content }),
      (err)     => ({ tag: 'planDocError',  taskId: action.taskId, message: err }),
    ),
  );
}
case 'closeDetail': {
  $.state.detailTaskId = null;
  $.state.planDoc = { tag: 'idle' };
  return null;
}
case 'planDocLoaded': {
  if ($.state.planDoc.tag === 'loading' && $.state.planDoc.taskId === action.taskId) {
    $.state.planDoc = { tag: 'loaded', taskId: action.taskId, content: action.content };
  }
  return null;
}
case 'planDocError': {
  if ($.state.planDoc.tag === 'loading' && $.state.planDoc.taskId === action.taskId) {
    $.state.planDoc = { tag: 'error', taskId: action.taskId, message: action.message };
  }
  return null;
}
```

### New env method

```typescript
export interface LedgerEnv {
  // ... existing ...
  fetchPlanDoc(taskId: number): Effect<Result<string, string>>;
}
```

`liveLedgerEnv` implementation fetches `GET /api/ledger/tasks/${taskId}/plan` and returns
`result.content` on 200 or an error message otherwise.

`mockLedgerEnv` returns `Effect.of(() => { send(Result.ok('# Mock Plan\n\nNo content.')); })`.

### Keyboard wiring in LedgerTable

- `Enter` on a selected row → `send({ tag: 'openDetail', taskId: selected.id })`
- `Escape` → `send({ tag: 'closeDetail' })`

### New API endpoint

```
GET /api/ledger/tasks/:id/plan
→ 200 { content: string }    // raw Markdown
→ 404 { error: string }      // task not found or file missing
```

Route definition added to `ledgerRouter` in `apps/api/src/ledger/router.ts`.

Handler reads `plan/{task.file}` relative to the workspace root. The plan directory is
resolved using `PLAN_DIR` env var (default: `path.resolve(process.cwd(), 'plan')`).

```

├─────────────────────────────────────────┤

│  ...                                     │
└─────────────────────────────────────────┘
```

- Drawer width: `min(520px, 100vw)`.
- Markdown rendered with `react-markdown` + `remark-gfm` + `rehype-highlight`.
- Loading state: `<CircularProgress>` centred in the drawer body.
- Error state: `<Alert severity="error">`.
- Code blocks get a dark surface background via `sx` on the `<pre>` element (no external
  CSS file — stays in-component via `components` prop on ReactMarkdown).

## Goal

1. Add `detailTaskId`, `planDoc` to `LedgerState` and `PlanDocState` type in
   `ledger.store.ts`.
2. Add four new actions to `LedgerAction` and handle them in `ledgerReducer`.
3. Add `fetchPlanDoc(taskId: number)` to `LedgerEnv` interface and implement in
   `liveLedgerEnv`.
4. Add `fetchPlanDoc` no-op mock to `mockLedgerEnv`.
5. Add `Enter` / `Escape` key handlers to `LedgerTable.tsx`.
6. Create `LedgerDetailDrawer.tsx` — reads `detailTaskId` + `planDoc` from `$.state`,
   dispatches `closeDetail` on the close button and on Escape.
7. Add the `GetTaskPlan` route to `ledgerRouter` in `apps/api/src/ledger/router.ts` and a
   `readPlanDoc(taskId, planDir)` function to `apps/api/src/ledger/reader.ts`.
8. Write `LedgerDetailDrawer.test.tsx` — tests listed in Success Criteria.
9. Update `ledger.store.test.ts` — cover `openDetail`, `closeDetail`,
   `planDocLoaded`, `planDocError` reducer cases.
