# PLAN.42 — Edge and corner case tests

## Context

The current test suite covers the happy path and obvious error cases well.
But the routing and type machinery has several structural edge cases that are
not systematically tested: empty trees, single-node trees, maximum nesting
depth, ambiguous path segments, wildcard collisions, and schema-less routes.
These are the cases most likely to surface subtle inference or runtime bugs
as the codebase evolves.

This plan is a focused test-writing exercise — no new runtime code.

## Goal

Add a dedicated `src/core/edge-cases.test.ts` (and
`src/server/edge-cases.test.ts` for server-side) covering the categories
below. Each test must either assert a correct runtime result or use
`@ts-expect-error` / `expectTypeOf` to assert a correct type-level behavior.

### Categories

**Empty and degenerate trees**
- `defineRoutes([])` — empty router; `parse('/')` returns `{ ok: false }`.
- Single leaf route; `print()` and `parse()` round-trip.
- Route with no schema (`schema: null`); body is `never` in handler.

**Path segment edge cases**
- Trailing slash vs. no trailing slash — consistent behavior.
- Consecutive path params: `/a/:x/:y/` — both typed.
- Wildcard (`*`) at root vs. wildcard in nested position.
- Path that is purely a wildcard: `*` — matches everything.
- Param name collision: parent `:id` and child `:id` — which wins?

**Schema edge cases**
- Route with `z.object({})` (empty schema) — `params` is `{}`, not `never`.
- Route schema with optional fields — fields absent from URL default correctly.
- Deeply nested optional fields in query schema.

**Composition**
- Two routers with a shared path prefix — disambiguation behavior.
- Composed router where one sub-router is empty.
- Route tag uniqueness: two routes with the same tag in different branches —
  document current behavior (allowed or error).

**Type-level only**
- Confirm `InferRoute<typeof router>` is `never` for an empty router.
- Confirm discriminated union is correctly widened when composing two routers.

## Change Surface

- New `src/core/edge-cases.test.ts`
- New `src/server/edge-cases.test.ts`
- No changes to runtime code. If a test exposes a real bug, open a new plan.

## Dependencies

- Plan 26 (type-level test coverage) — establishes patterns and helpers for
  `expectTypeOf` / `@ts-expect-error` tests; extend those patterns here.

## Success Criteria

- At least 30 new test cases across the two files.
- Every test case is in a named category (use `describe`) so failures are easy
  to locate.
- No existing tests are modified.
- `npm test && npm run typecheck` clean.
