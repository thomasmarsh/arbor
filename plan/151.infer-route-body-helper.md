# Plan 151 — `InferRouteBody<Router, Tag>` Type Helper

## Context

When the UI derives response types from the router contract (as in Plan 143), it must
drill through `_ctxMap` manually:

```typescript
type QueueCtx = LedgerRouter['_ctxMap']['ledger-get-queue'];
type DisplayGroupsResponse = InferSingleSuccessBody<QueueCtx['response']>;
```

This is the correct approach (router as source of truth) but the two-step lookup is
ergonomically painful and will appear at every call-site that needs a typed body.

## Goal

Export a single `InferRouteBody<Router, Tag>` utility type from `@arbor/router` that
collapses the two-step lookup into one:

```typescript
type DisplayGroupsResponse = InferRouteBody<LedgerRouter, 'ledger-get-queue'>;
```

## Change Surface & T-shirt Sizing

**Size: XS**

One file modified:
- `packages/router/src/contexts/http-context.ts` (or a new `types.ts`) — add
  `InferRouteBody<Router, Tag>`
- `packages/router/src/index.ts` — re-export `InferRouteBody`
- `packages/ui/src/ledger/ledger.store.ts` — replace the two-step lookup with the helper
