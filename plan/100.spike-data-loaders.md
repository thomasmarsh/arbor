# PLAN.100 (Spike) — Data Loaders Architecture

## Context

Browser navigation should trigger data loading before or during component mounting.
Two design dimensions need a decision before implementation:

**Dimension 1 — Where does the loader live?**

- **Route-attached**: `httpRoute(...).withLoader(async ({ id }) => fetchUser(id))` — loader
  is part of the route definition. Ergonomic; co-located with the schema it types.
  Risk: couples a fetch concern to `@arbor/router` which is supposed to be server-agnostic.
  The loader would need to be stripped from the server-side bundle.

- **Separate map**: `createLoaderMap(router, { user: async ({ id }) => fetchUser(id) })` —
  a parallel object keyed by route tag, typed by the route's param types. Explicit; no
  changes to `@arbor/router`. Risk: two structures to keep in sync.

**Dimension 2 — Suspense vs. explicit loading state?**

- **Suspense**: loaders throw Promises; React Suspense catches them; `<Suspense fallback>` in
  the component tree handles the loading UI. Standard React 18+ pattern. Works well for
  initial load; awkward for transitions (Suspense can cause flash-of-fallback).

- **Explicit state**: `useRoute()` returns `{ status: 'pending' | 'committed' | 'error', route, data }`.
  More explicit; no fallback flash; pairs naturally with `useTransition`. More boilerplate.

- **Both**: use `startTransition` internally so the current route stays visible during the
  transition, and expose `isPending` from `useRoute()`.

## Goal

Answer both dimensions with a minimal prototype. Write a scratch file
(`packages/router-browser/scratch/loaders-spike.ts`) that:

1. Implements a `createLoaderMap` approach with typed params.
2. Implements a `routeWithLoader` approach.
3. Tests both for type fidelity — does the loader param type infer correctly from the
   route schema?
4. Implements a minimal Suspense-compatible loader (throws Promise) and an explicit-state
   loader. Measure which has cleaner type signatures.

## Deliverable

- Suspense vs. explicit state?
- Recommended API shape for plan 101.
