# Plan 159 — Vitest Config Consolidation

## Context

Every workspace package ships its own `vitest.config.ts`. Most are trivial wrappers that only set `environment`, `include`, and occasionally `root`. They exist because Vitest requires a config file if you reference the package in `vitest.workspace.ts` by glob. With Vitest 2+'s inline project definitions, these files can be replaced by a `{ extends, test: { ... } }` object directly inside `vitest.workspace.ts`.

Current configs and their complexity:

| File | Unique settings | Action |
|------|----------------|--------|
| `framework/router/vitest.config.ts` | `environment: node`, `include`, `root` | **inline** |
| `framework/common/vitest.config.ts` | `exclude: dist/**` only | **inline** |
| `framework/router-browser/vitest.config.ts` | `environment: jsdom`, `include`, `root` | **inline** |
| `apps/bff/vitest.config.ts` | `environment: node`, `include` | **inline** |
| `apps/ui/vitest.config.ts` | `environment: jsdom` | **inline** |
| `apps/api/vitest.config.ts` | `environment: node`, `include`, **`setupFiles`** | **keep** — `setupFiles` path resolves relative to the config file; inlining breaks the path |
| `framework/router-test/vitest.config.js` | path `alias` to `../router/src` | **keep** — `resolve.alias` is non-trivial |

Net result: 5 files deleted, `vitest.workspace.ts` grows by ~30 lines, workspace root stays the single place to understand the test topology.

## Goal

Collapse the five simple per-package vitest configs into inline project definitions in `vitest.workspace.ts`. Delete the five source files. The test suite must pass identically before and after.

## Change Surface & T-shirt Sizing

**S** — one file rewritten, five files deleted, no logic changes.

| File | Change |
|------|--------|
| `vitest.workspace.ts` | Replace five glob-match entries with inline `{ extends, test }` objects |
| `framework/router/vitest.config.ts` | Delete |
| `framework/common/vitest.config.ts` | Delete |
| `framework/router-browser/vitest.config.ts` | Delete |
| `apps/bff/vitest.config.ts` | Delete |
| `apps/ui/vitest.config.ts` | Delete |

### Inline definition shape

```ts
{
  extends: './framework/router/package.json', // or vite.config.ts for UI
  test: {
    name: '@arbor/router',
    root: './framework/router',
    environment: 'node',
    include: ['src/**/*.test.ts'],
  },
},
```

For `@arbor/ui` the `extends` field should point at `./apps/ui/vite.config.ts` so Vite plugin resolution (React, basicSsl) is inherited correctly.

## Dependencies

None.

## Success Criteria

- `pnpm test` produces identical pass/fail counts before and after.
- None of `framework/router/vitest.config.ts`, `framework/common/vitest.config.ts`, `framework/router-browser/vitest.config.ts`, `apps/bff/vitest.config.ts`, `apps/ui/vitest.config.ts` exist on disk after the change.
- `apps/api/vitest.config.ts` and `framework/router-test/vitest.config.js` are unchanged.
