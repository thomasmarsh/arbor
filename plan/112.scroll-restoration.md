# PLAN.112 — Scroll Restoration

## Context

When navigating back/forward in a SPA, users expect scroll position to be restored to
where they were on that page. Browsers do this natively for hard navigations; SPAs using
`history.pushState` must implement it manually.

## Goal

1. On every navigation away from a route, save `window.scrollY` (and any scrollable
   container positions) keyed by the serialized route.
2. On navigation to a route that was previously visited (back/forward), restore the saved
   scroll position after the new route commits (after loaders resolve, after DOM paint).
3. On navigation to a new route (forward navigation to a never-visited route), scroll to
   the top.
4. Per-route configuration via `scrollBehavior`:
   - `'top'` — always scroll to top (default for new navigations).
   - `'restore'` — restore saved position (default for back/forward).
   - `'preserve'` — do not change scroll (useful for modal-like routes).
   - `(route) => { x: number; y: number }` — custom scroll target.

## API Surface

```typescript
// BrowserRouter config
const browser = new BrowserRouter(router, {
  scrollBehavior: 'restore', // global default
});

// Per-route override in meta map
const metaMap = createMetaMap(router, {
  modal: () => ({ scrollBehavior: 'preserve' }),
});
```

## Success Criteria

- `window.scrollY` is saved on navigation away.
- Back/forward navigation restores saved scroll position after DOM commit.
- New forward navigations scroll to top.
- `scrollBehavior: 'preserve'` leaves scroll unchanged.
- Custom function receives typed route params.
