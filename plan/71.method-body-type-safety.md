# Plan 71 — Method/body type safety

## Context

`spec.enhancements.md` Phase 1 #3: "Enforce at the type level that if a context uses `GET`,
it cannot accept a `body` payload, while `POST`/`PUT`/`PATCH` mandate it if specified."

Currently `httpRoute(schema, 'GET', 'items', { body: z.string(), response: ... })` compiles
with no error. GET, HEAD, and DELETE requests carry no semantic body per RFC 7231 §4.3;
accepting a body schema on those methods is a contract mistake that the type system should
catch immediately.

The second half of the spec item — mandating body on POST/PUT/PATCH — is more contentious:
legitimate action endpoints (e.g., `POST /logout`) carry no body. This plan addresses only
the prohibition on idempotent read methods.

## Goal

`httpRoute()` produces a compile-time error if `body` is provided when the method is `GET`,
`HEAD`, or `DELETE`.

## Approach

Discriminate the options type on `Method`:

```ts
type SafeBodyOption<Method extends HttpMethod> =
  Method extends 'GET' | 'HEAD' | 'DELETE' ? { body?: never } : { body?: z.ZodType };

// merged into the options parameter of httpRoute()
```

`body?: never` is the canonical TypeScript idiom: the key can be omitted but, if present,
its value must be `never` — which no runtime value satisfies. TypeScript reports an error at
the call site the moment a real `body` value is passed.

The `openApiRoute()` wrapper delegates to `httpRoute()` so it inherits the constraint at
no extra cost.

## Change Surface

- `src/contexts/http-context.ts` — modify the `options` type in `httpRoute()`'s signature
- `src/contexts/http-context.test.ts` — add `expectTypeOf` / `@ts-expect-error` tests
  confirming GET+body is a type error and POST+body is valid
- `src/contexts/openapi-context.test.ts` — same

## Dependencies

None. Pure type-level change; zero runtime impact.

## Success Criteria

- `httpRoute(s, 'GET', 'p', { body: z.string(), response: {} })` is a type error.
- `httpRoute(s, 'DELETE', 'p', { body: z.string(), response: {} })` is a type error.
- `httpRoute(s, 'POST', 'p', { body: z.string(), response: {} })` compiles cleanly.
- `httpRoute(s, 'POST', 'p', { response: {} })` (no body) still compiles cleanly.
- `pnpm test && pnpm typecheck` green.
