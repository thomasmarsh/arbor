# PLAN.96 — Typed URL Search Params

## Context

`httpRoute` already accepts a `query` Zod schema for server-side query-param validation.
In the browser, we want the same schema to drive `parseSearch` (URL → typed object) and
`printSearch` (typed object → URL search string). This avoids maintaining two separate
query-schema registries (one for the server, one for the browser) and ensures the browser
and server agree on query-param shapes.

There is a subtlety: the server `query` schema is already part of `HttpContext._meta`. The
browser needs to access it without importing server-only code. The `RouterContract` is the
right interface — it already carries the context map which includes the `query` type.

## Goal

1. `parseSearch<Tag>(url, router, tag)` — extract and parse the search string from a URL
   using the Zod schema registered for `tag`; returns `Result<QueryType, ZodError>`.
2. `printSearch<Tag>(params, router, tag)` — serialize typed query params to a
   `URLSearchParams`-compatible string.
3. Make `BrowserRouter.navigate(route, search?)` accept optional typed search params
   for routes that declare a query schema.

## API Surface

```typescript
export function parseSearch<
  Tag extends keyof Map & string,
  Map extends AnyCtxMap,
>(url: string | URL, router: RouterContract<any, Map>, tag: Tag): Result<z.infer<Map[Tag]['query']>, z.ZodError>;

export function printSearch<
  Tag extends keyof Map & string,
  Map extends AnyCtxMap,
>(params: z.infer<Map[Tag]['query']>, router: RouterContract<any, Map>, tag: Tag): string;
```

## Open Question

Routes without a `query` schema should have `query: never` in their context type — calling
`parseSearch` / `printSearch` on them should be a compile-time error. Confirm that the
current `HttpContext` type already expresses this, or add a `query: never` default.

## Success Criteria

- `parseSearch` returns correctly typed object for a route with a known `query` schema.
- `printSearch` produces a `?key=val&...` string that round-trips through `parseSearch`.
- Calling either function on a route with no query schema is a type error.
