# PLAN.103 — Optimistic Prefetch

## Context

When a user hovers over a `<Link>`, the target route's loaders can fire in the background.
If the user then clicks, the data is already cached and navigation is instant. This is the
same pattern as TanStack Router's `preload` prop and Next.js `prefetch`.

The loader cache (introduced in plan 101) is the mechanism: prefetch populates the cache
under the same key as a real navigation would, so the actual navigation finds data ready.

## Goal

1. `<Link to={route} prefetch="hover">` — fires loaders on `mouseenter`; cancels on
   `mouseleave` if navigation hasn't started (avoids wasted requests on quick hover).
2. `<Link to={route} prefetch="render">` — fires loaders immediately when the link
   renders (e.g., above-the-fold links in a list).
3. `prefetch` defaults to `"hover"`.
4. Loader cache TTL: prefetched entries expire after a configurable duration
   (default 30 s) to avoid stale data from a long hover.

## API Surface

```typescript
<Link to={{ tag: 'user', id: '42' }} prefetch="hover">View Profile</Link>
<Link to={{ tag: 'user', id: '42' }} prefetch="render">View Profile</Link>
<Link to={{ tag: 'user', id: '42' }} prefetch="none">View Profile</Link>
```

## Success Criteria

- Hovering a link with `prefetch="hover"` triggers loader execution.
- Navigating after a prefetch does not re-execute loaders (cache hit).
- Quick hover-and-leave (`mouseleave` before click) cancels in-flight prefetch.
- Prefetch entries expire after TTL; stale entries are not served.
