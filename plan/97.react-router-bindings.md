# PLAN.97 — React Router Bindings (`packages/router-react`)

## Context

With `BrowserRouter` in place, React bindings are a thin layer: a context provider holds
the `BrowserRouter` instance and the current route; hooks subscribe to navigation updates;
components re-render when the route changes. The goal is minimal surface area — just enough
for a component switch to work.

## Goal

Create `packages/router-react` with:

1. `RouterProvider` — context provider; wraps `BrowserRouter`; subscribes to navigation;
   re-renders children when route changes.
2. `useRoute()` — returns `Result<Route, string>` for the current URL; reactive.
3. `useNavigate()` — returns `(route: Route) => void`.
4. `useMatch(tag)` — returns the narrowed route object if the current route's tag matches,
   else `null`. Useful for conditional rendering without a full switch.

## API Surface

```typescript
// RouterProvider
<RouterProvider router={browserRouter}>
  {children}
</RouterProvider>

// useRoute — returns the current parsed route
const route = useRoute<Route>(); // Result<Route, string>

// useNavigate — typed navigation
const navigate = useNavigate<Route>();
navigate({ tag: 'user', id: '42' }); // pushState + rerender

// useMatch — tag-narrowed access
const userRoute = useMatch('user'); // { tag: 'user', id: string } | null
```

## Success Criteria

- `useRoute()` reflects navigation after `navigate()` is called.
- `useMatch('user')` returns the narrowed object when active, `null` otherwise.
- `RouterProvider` cleans up the `BrowserRouter` subscription on unmount.
- All tests pass under `jsdom`.
- No `any` in the public hook signatures.

## Notes

React context generic parameter: `RouterContextValue<Route>` must carry the `Route` type
parameter. This requires a generic context factory (same pattern as React Query's
`QueryClientContext`). Do not use a single global context — there must be one context
instance per `BrowserRouter` instance to support nested test isolation.
