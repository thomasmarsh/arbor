# PLAN.47 — Remove `_child` phantom via depth-counter `FlattenChildrenImpl`

## Context

Plan 28 was blocked because removing `_child` forced `Derive`/`ChildUnion` into
mutual recursion, hitting TS2589 even on 3-level trees. Plan 46 spike proved
that a depth-counter approach eliminates the mutual recursion without exposing
the counter in public API signatures.

**Spike result (Green):** `FlattenChildrenImpl<C, D>` — a single self-recursive
mapped type with an internal depth counter — passes `tsc --noEmit` clean for all
existing test cases plus a 4-level tree. Public aliases `NewDerive` and
`NewChildUnion` have the same signatures as the originals; the depth counter is
invisible to callers.

## Goal

Replace `Derive` + `ChildUnion` (which rely on the `_child` phantom for O(1)
child-union recovery) with `FlattenChildrenImpl` (which recurses through the
`children` array directly), then delete `_child` from `RouteNode`.

## Change Surface

1. **`src/core/route-node.ts`**
   - Add `Prev`, `ValidDepth`, `PrevD`, and `FlattenChildrenImpl` types.
   - Replace `Derive<N>` body: `FlattenChildrenImpl<[N]>[0]`
   - Replace `ChildUnion<C>` body: `FlattenChildrenImpl<C>[number]`
   - Remove `_child: Child` from the `RouteNode` interface.
   - Remove the `Child` type parameter from `RouteNode` (it becomes derivable from `children`).

2. **`src/core/define-routes.ts`**
   - Remove every `_child: undefined as never` assignment.

3. **`src/core/route-node.test.ts`**
   - Confirm all `Derive` and `ChildUnion` assertions still pass unchanged.
   - Remove any assertions that reference `_child`.

4. Run verification chain after each step.

## Dependencies

- Plan 46 spike complete. ✓
- Plan 19 complete (stable `RouteNode` baseline). ✓

## Success Criteria

- `RouteNode` has no `_child: Child` field and no `Child` type parameter.
- `Derive` and `ChildUnion` have identical public signatures to before.
- All existing tests pass without modification.
- `pnpm test && pnpm typecheck && pnpm lint` green.
- Net deletion of lines.

## Trade-offs / Notes

- `FlattenChildrenImpl` supports up to 15 levels of nesting (depth counter
  default = 15). This is configurable by expanding the `Prev` tuple and
  `ValidDepth` union. In practice, route trees deeper than 10 are unusual.
- The `_child` removal is still a pure win: less noise in autocomplete and
  docs, and the depth counter adds negligible cost (TypeScript resolves
  `Prev[D]` in O(1) via tuple index).
