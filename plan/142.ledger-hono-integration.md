# Plan 142 — Hono Integration & ledgerRouter Export

## Context

The existing `packages/api/src/index.ts` uses Hono as its HTTP server with two route groups
(`/api/hello`, `/api/users`). This plan mounts the arbor ledger handler alongside those routes
and exports `ledgerRouter` so `packages/ui` can use it for `createClient` type inference.

## Goal

- `GET /api/ledger/tasks`, `GET /api/ledger/queue`, `GET /api/ledger/tasks/:id` respond correctly
  when `pnpm --filter @arbor/api dev` is running.
- `packages/api/src/index.ts` re-exports `ledgerRouter` and `typeof ledgerRouter`.

## Change Surface & T-shirt Sizing

**Size: XS**

Modified files:
- `packages/api/src/index.ts` — add Hono catch-all + re-export
- `packages/api/package.json` — add `@arbor/router` dependency (if not already added in 141)

## Dependencies

- Plan 141 complete (`ledgerServer`, `ledgerRouter` exist)

## Implementation

```typescript
// packages/api/src/index.ts  (additions only)
import { ledgerServer, ledgerRouter } from './ledger/routes.js';

// Mount before the catch-all, after logger middleware
app.all('/api/ledger/*', async (c) => {
  const body = await c.req.json().catch(() => undefined);
  const result = await ledgerServer.handle(
    new URL(c.req.url),
    c.req.method,
    body,
  );
  return c.json(result.body, { status: result.status });
});

// Re-export router type for packages/ui typed client
export { ledgerRouter };
export type LedgerRouter = typeof ledgerRouter;
```
