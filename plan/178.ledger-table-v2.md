# Plan 178 — LedgerTable v2: MUI Rich Table

## Context

The current `LedgerTable.tsx` renders a bare HTML `<table>` with inline styles. The
`selectedIndex` highlight is `background: '#264f78'`, done tasks use `opacity: 0.5`, and
pressing ArrowUp/ArrowDown both moves the cursor **and** scrolls the page — classic scroll
jank because the keydown handler doesn't call `preventDefault()`.

This plan replaces the table with MUI components and fixes the jank. The reducer, env, and
all existing actions are **unchanged** — the component is purely presentational.

### Visible improvements

- **Status chips** — color-coded MUI `<Chip>` per status:
  `next` = blue, `in_progress` = amber, `todo` = grey, `done` = green, `canceled` = grey
- **Kind icons** — science/spike icon for `kind: 'spike'`; checklist for `kind: 'task'`
- **Group separator rows** — labelled `TableRow` dividers between _In Progress_, _Ready_,
  and _Blocked_ sections (and _Done_ when showAll is true)
- **Selected row** — MUI `<TableRow selected>` (sets `aria-selected="true"`) instead of
  inline background
- **Dimmed done/canceled** — `sx={{ opacity: 0.4 }}` on the row rather than inline style
- **Keyboard shortcut bar** — a compact `<Typography variant="caption">` footer listing
  active shortcuts (j/k n d b D a r) so the UI is self-documenting
- **Arrow key jank fix** — `e.preventDefault()` for `ArrowUp` and `ArrowDown` in the
  keydown handler so the window does not scroll

### New columns

Add **Epic**, **Story**, and **Layer** columns (compact, monospace `<Chip size="small">`).
Remove nothing — keep ID, Wave, Status, Size, Task.

The table header row is sticky (`stickyHeader` on `<Table>`).

## Goal

1. Replace `LedgerTable.tsx`'s `<table>` with MUI `Table / TableHead / TableBody /
   TableRow / TableCell` components. No logic changes — same reducer, same `send()` calls,
   same `useStore` / `useEffect` structure.
2. Add `e.preventDefault()` for `ArrowUp` and `ArrowDown` in the keydown `useEffect`.
3. Render group separator rows. Helper: `GroupRow({ label }: { label: string })` — a
   `<TableRow>` with a single `<TableCell colSpan={8}>` containing a muted label.
4. Render `StatusChip` — a thin wrapper around `<Chip>` that maps `TaskStatus` to a color.
   Keep it in the same file or a co-located `LedgerChips.tsx`; no separate export from
   `index.ts`.
5. Update `LedgerTable.test.tsx`:
   - Replace `style` attribute assertions with `aria-selected="true"` on the selected row.
   - Assert group separator rows are present (query by text "In Progress", "Ready").
   - Assert arrow key events have `preventDefault` called — use
     `userEvent.keyboard('{ArrowDown}')` and confirm no window scroll (spy on
     `window.scrollBy` or simply assert `aria-selected` moves without additional
     `scrollY` change).

2. `pnpm --filter @arbor/ui typecheck` passes.
3. The selected row has `aria-selected="true"` (verifiable in tests).
4. Group separator rows render with labels matching the section name.

6. Status chips render with the correct `color` prop for each status value.
