# Plan 147 — Keyboard Shortcuts, Row Selection & Show-All Toggle

## Context

The store can mutate tasks (Plan 146). This plan wires keyboard interactions into the
`LedgerTable` component to make it operate like the Python TUI. Row selection state is already
in `LedgerState.selectedIndex`. We also add the show-all toggle here since it's a one-liner
state change in the same component.

## Goal

The LedgerTable responds to keyboard shortcuts matching the Python TUI bindings:

| Key | Action |
|-----|--------|
| `n` | toggle selected task: todo ↔ next |
| `d` | toggle selected task: todo ↔ done |
| `b` | bump (move to front of wave) |
| `D` | defer (move to back of wave) |
| `a` | toggle show-all (active only ↔ all tasks) |
| `r` | re-fetch |
| `↑` / `k` | move selection up |
| `↓` / `j` | move selection down |

## Keyboard Handler

```tsx
useEffect(() => {
  const onKey = (e: KeyboardEvent) => {
    if (e.target instanceof HTMLInputElement) return;
    const rowCount = visibleRows.length;
    switch (e.key) {
      case 'ArrowUp': case 'k': send({ tag: 'selectUp' }); break;
      case 'ArrowDown': case 'j': send({ tag: 'selectDown', rowCount }); break;
      case 'n': send({ tag: 'setStatus', taskId: selected.id, status: toggleNext(selected.status) }); break;
      case 'd': send({ tag: 'setStatus', taskId: selected.id, status: toggleDone(selected.status) }); break;
      case 'b': send({ tag: 'bump', taskId: selected.id, waveRanks: waveRanksFor(selected.wave) }); break;
      case 'D': send({ tag: 'defer', taskId: selected.id, waveRanks: waveRanksFor(selected.wave) }); break;
      case 'a': send({ tag: 'toggleShowAll' }); break;
      case 'r': send({ tag: 'refresh' }); break;
    }
  };
  window.addEventListener('keydown', onKey);
  return () => window.removeEventListener('keydown', onKey);
}, [send, selected, visibleRows]);
```

`toggleNext` / `toggleDone` mirror the Python toggle logic (toggle to that status if not already
there, else revert to `todo`).

## Row Highlighting

Selected row gets a CSS class. The component scrolls the selected row into view using a `ref`
on each `<tr>` and calling `scrollIntoView({ block: 'nearest' })` when `selectedIndex` changes.

## Show-All Toggle

When `showAll` is false (default), only inProgress + ready + blocked rows are shown.
When `showAll` is true, done and canceled rows are appended (dimmed via CSS opacity).

The `/api/ledger/queue` response already includes done and canceled — no new API route needed.

## Success Criteria

- Arrow keys move the highlighted row.
- `n` / `d` toggle the selected task's status and the table re-renders immediately.
- `b` / `D` reorder within the wave.
- `a` shows/hides done tasks.
- Wave 28 slice is complete: full mutation loop works end-to-end from keyboard to disk.
