# 182. Spike — DnD Framework Integration Pattern

## Context

Column reordering (plan 183) and potentially task reordering require drag-and-drop. Our component pattern wraps state in a TCA `useStore` + `send` idiom. DnD libraries expose callbacks (`onDragEnd`, `onDrop`) that produce imperative results; we need to bridge those into `send(action)` cleanly, without leaking library internals into store action shapes.

The main design question: what does the minimal adapter surface look like so that DnD usage sites only need to express "when item A drops onto position B, send this action" — with no DnD library types appearing at the call site?

Candidate libraries:
- `@dnd-kit/core` + `@dnd-kit/sortable` — headless, works cleanly alongside MUI, actively maintained
- `react-beautiful-dnd` — good UX defaults but deprecated; large bundle; known MUI conflicts
- Native HTML5 DnD — no dependency but poor touch support and janky default visuals

## Goal

1. Choose a library with rationale.
2. Design a minimal adapter hook: something like `useSortable<K>(items: T[], getId: (t: T) => K, onReorder: (fromId: K, toId: K) => void)` that returns MUI-compatible drag handle props and a sortable container ref — caller wires `onReorder` to `send(...)`.
3. Prototype the hook against a sortable list wired to a test store; verify it compiles and runs.
4. Commit a design note to `plan/` summarizing the chosen pattern and any constraints discovered.

## Success Criteria

- A working prototype of a sortable list that calls `send(reorderAction)` on drop.
- A clear library choice with rationale (bundle size, MUI compatibility, touch support, TypeScript quality).
- A draft API for the generic adapter (hook signature, types, constraints).
- Any discovered footguns documented (e.g., React strict-mode double-render, keyboard accessibility needs).
- Plan 183 can be written from the outputs of this spike without further unknowns.
