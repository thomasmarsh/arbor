# Plan 84 ‚Äî Framework-internal property-based testing

## Context

Plan 79 (`@arbor/router-test`) describes a user-facing PBT product: `createRouteTests`
that generates tests for *adopters'* handlers. This plan is separate and earlier: it
adds `fast-check` to the router's own `devDependencies` and tests the **framework's
own core invariants**.

The segment parser, walk algorithm, and print/parse round-trip are pure functions with
strong mathematical properties. Example-based unit tests cover specific inputs; PBT
covers the space between them ‚Äî the `undefined` field, the integer boundary, the
deeply-nested tree that exhausts the depth counter.

This plan should land before Plan 79. Plan 79 builds on the same `fast-check`
installation and depends on these invariants being proven first.

## Goal

Add `fast-check` as a devDependency; add PBT tests in three layers:

1. **`segments.test.ts`**: `parseSegments` never-throw + structural invariants.
2. **`walk.test.ts`**: `walkParse` never-throw + `print`/`parse` round-trip.
3. **`edge-cases.test.ts`** (core): adversarial inputs that should never crash.

## Invariants to encode

### Layer 1 ‚Äî Segment parser

- `parseSegments(anyString)` never throws for any non-empty string input
- Every returned segment has a non-empty `name` (params) or `value` (literals)
- `parseSegments(s).length >= 1` for any non-empty string input (at minimum one segment)

### Layer 2 ‚Äî Walk / print / parse

Round-trip:

```
// For any valid route in the tree:
parse(print(route)) deepEquals route
```

Never-throw (the core safety guarantee):

```
// For any URL string:
router.parse(new URL(anyUrl)) does not throw ‚Äî it returns Err(diag) or Ok(route)
```

Monotonicity:

```
// parse always returns one of: Ok(route where tag ‚àà known tags) | Err(ParseDiag)
// It never returns Ok with a tag that isn't in the route tree
```

### Layer 3 ‚Äî Adversarial edge inputs

- URL with empty path segment (`//`)
- URL with very long path (> 1000 characters)
- URL with all segment kinds in one path (`/users/:id/#num/*rest`)
- Query string with special characters (`?a=<>&b="`)

## Arbitrary generators needed

```typescript
// A valid route object from a known tree ‚Äî generated by walking the tree
// and randomly selecting a tagged node, then filling in required params.
function validRouteArb<R>(router: { children: ...; _type: R }): fc.Arbitrary<R>

// A URL path string (may or may not match any known route)
const anyPathArb: fc.Arbitrary<string> = fc.webPath()
```

The `validRouteArb` helper is straightforward: collect all tagged nodes via
`forEachTaggedNode`, generate random string/number values for params matching each
segment kind, and construct the route object.
