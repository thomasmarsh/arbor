# Plan 152 — Recursive CtxMap for Section-Nested Routes

## Context

`CtxMap<C>` in `route-node.ts` iterates only `C[number]` — the direct children of the
array passed to `defineRoutes`. Section nodes (`schema: null`, `R = never`) are filtered
out by the `N extends RouteNode<{ tag: infer T … }>` guard and their children are never
visited. This means any route nested inside a `section()` node is invisible to
`createServer`: the handler map would be typed as `{}` and the handler keys would be
unknown to TypeScript.

`FlattenChildrenImpl` in the same file already handles section recursion correctly for
the *route-type union* (the `_type` phantom). This plan brings `CtxMap` to parity.

## Goal

`CtxMap<C>` recurses through section nodes so that any `httpRoute` at any depth in
the tree contributes to the handler map typing. After this plan, the following
compiles correctly and produces a fully-typed handler map:

```typescript
const router = defineRoutes([
  section('api', [
    httpRoute(HelloRoute,  'GET', 'hello', { response: { 200: HelloResponseSchema } }),
    section('ledger', [
      httpRoute(GetTasks, 'GET', 'tasks', { response: { 200: TasksResponse } }),
    ]),
  ]),
]);

createServer(router, {
  hello:            async (_ctx) => respond(200, { … }),
  'ledger-get-tasks': async (_ctx) => respond(200, { … }),
});
```

## Change Surface & T-shirt Sizing

**Size: S**

Single file change:

- `packages/router/src/core/route-node.ts` — rewrite `CtxMap<C>`

### Design

The current definition:

```typescript
export type CtxMap<C extends RouteNode<…>[]> = {
  [N in C[number] as N extends RouteNode<{ tag: infer T … }, …> ? T : never]:
    N extends RouteNode<any, any, infer Ctx, …> ? Ctx : never;
};
```

New definition — recurse into section children using the same depth-counter
machinery already present for `FlattenChildrenImpl`:

```typescript
type CtxMapDeep<C extends RouteNode<…>[], D extends ValidDepth = 14> =
  UnionToIntersection<CtxMapEntry<C[number], D>>;

type CtxMapEntry<N extends RouteNode<…>, D extends ValidDepth> =
  [PrevD<D>] extends [never] ? Record<never, never> :
  N extends RouteNode<infer R, infer GC extends RouteNode<…>[], infer Ctx, any, any>
    ? [R] extends [never]
      ? CtxMapDeep<GC, PrevD<D>>           // section node → recurse
      : [R] extends [{ tag: infer T extends string }]
        ? Record<T, Ctx>                   // route node → contribute
        : Record<never, never>
    : Record<never, never>;

export type CtxMap<C extends RouteNode<…>[]> = CtxMapDeep<C>;
```

`UnionToIntersection` is a standard conditional-type helper — add it to
`route-node.ts` if not already available in the codebase.

The `Record<T, Ctx>` approach (intersection of single-key records) produces the same
object shape as the current mapped-type approach when there are no duplicate tags.
Duplicate tags are caught at runtime by `defineRoutes`'s tag-collision check.

### Backward compatibility

For a flat array of `httpRoute` nodes with no sections, `CtxMapDeep` returns the same
type as the current `CtxMap`. No existing tests or callsites require changes.
