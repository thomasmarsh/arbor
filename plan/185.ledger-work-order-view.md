# 185. Ledger Work-Order View

## Context

The "Ready" and "In Progress" groups show what could be started, but the list doesn't express recommended execution order. A work-order view would present a single flat ordered list — no group separators — where position encodes recommended priority: respecting wave sequence, "next" marker, dependency topological order, and rank.

The `computeDisplayGroups` function in `reader.ts` already sorts by `(waveOrder, rank)`. A work-order view extends this with topological ordering across the dependency graph.

## Goal

Add a third view mode: `'workOrder'`. Tasks appear in a single list in recommended execution order. Blocked tasks are shown (dimmed) with inline annotations listing their pending dependencies. Tasks marked `next` float to the top of their wave.

## Success Criteria

- Work-order view shows all non-canceled tasks in a single list, ordered by wave → next marker → topological dep order → rank.
- Blocked tasks appear at their natural position in the order, dimmed, with a "waiting on" annotation.
- Switching between all three view modes preserves active filters.
- New `computeWorkOrder` function has unit tests covering cycles (should not infinite-loop), `next` floating, and cross-wave ordering.
