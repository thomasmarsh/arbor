# Plan 92 — Capability System: Service Injection Across HTTP / SSE / WS Layers

## Context

Plan 80 (spike) validated all five questions affirmatively — no Effect-TS required.
The spike lives at `scripts/spike-capability.ts`. This plan delivers the full integration
across all three protocol layers.

**Core idea**: a route declares `needs: ['db:read', 'cache'] as const`. TypeScript narrows
the handler's `ctx.services` to exactly `Pick<ServiceRegistry, 'db:read' | 'cache'>`.
The server factory enforces at compile time that every declared service is provided.
`ServiceRegistry` is an augmentable interface — application code extends it via declaration
merging without modifying the library.

## Goal

1. Handlers on any protocol (HTTP, SSE, WS) can declare service dependencies via `needs`.
2. `ctx.services` is a typed `Pick<ServiceRegistry, Needs[number]>` — undeclared services
   are absent from the type.
3. Server factories (`createServer`, `createSseServer`, `createWsServer`) require that every
   needed service is present in `options.services` at compile time.
4. Existing routes that do not declare `needs` are unaffected (zero breaking changes).

## Design

### Phase 0 — `src/core/service-registry.ts`

```ts
// Augmentable registry — app code adds entries via declaration merging:
//   declare module '@arbor/router' { interface ServiceRegistry { 'db:read': DbReadService } }
// eslint-disable-next-line @typescript-eslint/no-empty-object-type
export interface ServiceRegistry {}

export type ServiceKey = keyof ServiceRegistry;

// ServicesCtx<Keys> is the `services` sub-object type injected into handler ctx.
export type ServicesCtx<Keys extends ServiceKey> = {
  services: Pick<ServiceRegistry, Keys>;
};

// Utility: slice a full services map to only the declared keys.
// Used internally by all server factories.
export function sliceServices<Keys extends readonly ServiceKey[]>(
  all: Pick<ServiceRegistry, Keys[number]>,
  needs: Keys,
): Pick<ServiceRegistry, Keys[number]> {
  return Object.fromEntries(
    needs.map((k) => [k, (all as Record<string, unknown>)[k]]),
  ) as Pick<ServiceRegistry, Keys[number]>;
}
```

#### `HandlerCtx` (server.ts)

Add `services` field with the same conditional pattern used for `session`:

```ts
export type HandlerCtx<
  CtxMap extends Record<string, HttpContext<HttpMethod, unknown, Record<number, unknown>, unknown, unknown, unknown, unknown, unknown>>,
  Routes,
  Tag extends keyof CtxMap & string,
> = {
  params: Omit<Extract<Routes, { tag: Tag }>, 'tag' | 'child' | 'query'>;
  body: CtxMap[Tag]['body'];
  query: CtxMap[Tag]['query'];
  headers: CtxMap[Tag]['headers'];
  cookies: CtxMap[Tag]['cookies'];
} & (CtxMap[Tag]['session'] extends never ? Record<never, never> : { session: CtxMap[Tag]['session'] })
  & (CtxMap[Tag]['services'] extends never ? Record<never, never> : { services: CtxMap[Tag]['services'] }); // ← new
```

#### `createServer` — service provision (server.ts)

Collect the union of all service keys across all routes in the `CtxMap`, then use a
conditional rest-parameter to make `options.services` required when any route has needs:

```ts
// Collects the union of all declared service keys across the full CtxMap.
type AllServiceKeys<M extends AnyCtxMap> = {
  [K in keyof M]: M[K] extends HttpContext<any, any, any, any, any, any, any, infer S>
    ? keyof S & ServiceKey
    : never;
}[keyof M];

type BaseServerOptions = {
  errorMap?: ErrorMapEntry[];
  maxBodySize?: number;
  rateLimitStore?: RateLimitStore;
  rateLimitKeyResolver?: RateLimitKeyResolver;
  resolveSession?: (headers: Record<string, string>) => Promise<SessionCtx | null>;
  onError?: (err: unknown, tag: string) => void;
  supervise?: boolean;
};

export function createServer<
  Route extends { tag: string },
  Map extends AnyCtxMap,
>(
  router: RouterContract<Route, Map>,
  handlers: HandlerMap<Map, Route>,
  ...args: AllServiceKeys<Map> extends never
    ? [options?: BaseServerOptions]
    : [options: BaseServerOptions & { services: Pick<ServiceRegistry, AllServiceKeys<Map>> }]
)
```

At runtime, `collectHttpMaps` now also returns a `needsMap: Record<string, readonly string[]>`.
Before calling a handler, the dispatch function slices the services:

```ts
const needed = needsMap[tag] as readonly ServiceKey[] | undefined;
const services = needed?.length
  ? sliceServices(options.services as Pick<ServiceRegistry, ServiceKey>, needed as ServiceKey[])
  : undefined;

const handlerCtx = { params, body, query, headers, cookies, ...(session ? { session } : {}), ...(services ? { services } : {}) };
```

## Success Criteria

1. `pnpm test && pnpm typecheck && pnpm lint && pnpm run examples` passes.
2. A handler with `needs: ['db:read']` has `ctx.services['db:read']` typed as the
   registered service; `ctx.services['db:write']` is a type error (`expectTypeOf` test).
3. `createServer(router, handlers, { /* services omitted */ })` is a type error when any
   route declares needs; `createServer(router, handlers)` is valid when no route has needs.
4. Runtime: `services` is sliced to declared keys before handler invocation.
5. SSE and WS layers follow the same pattern with their own `expectTypeOf` tests.
6. No existing tests are broken; no `as any` casts introduced.
