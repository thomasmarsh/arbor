# Plan 162 — `arbor` env config system

## Context

The current setup spreads environment config across six locations:

- `.env.mock`, `.env.bff`, `.env.oidc`, `.env.staging` — committed mode vars
- `apps/api/.env.local`, `apps/bff/.env.local`, `apps/bff/.env.staging.local` — gitignored local secrets

These feed into `dotenv-cli` invocations duplicated across five `dev:*` scripts and three `db:*` scripts in the root `package.json`. Adding an app means touching all of them.

## Goal

Collapse all env config into two files at the workspace root:

- **`arbor.toml`** (committed) — canonical mode definitions; replaces all `.env.*` mode files
- **`arbor.local.toml`** (gitignored) — machine-local secrets by app; replaces all `apps/*/env.local` files
- **`arbor.local.example.toml`** (committed) — setup template; replaces `apps/*/env.example` files

`arbor_cli` owns all env composition. `dotenv-cli` is removed from root deps entirely.

## Design

### `arbor.toml` (committed, workspace root)

```toml
[modes.mock]
VITE_AUTH_MODE = "mock"
ARBOR_AUTH_DISABLED = "true"

[modes.bff]
VITE_AUTH_MODE = "bff"
ARBOR_AUTH_DISABLED = "false"

[modes.oidc]
VITE_AUTH_MODE = "oidc"
ARBOR_AUTH_DISABLED = "false"
VITE_OIDC_ISSUER = "http://localhost:8080/realms/arbor"
VITE_OIDC_CLIENT_ID = "arbor-ui"
VITE_APP_URL = "http://localhost:5173"

[modes.staging]
VITE_AUTH_MODE = "bff"
ARBOR_AUTH_DISABLED = "false"
ARBOR_APP_URL = "https://localhost:5173"
VITE_USE_HTTPS = "true"
bff_profile = "staging"    # load [bff.staging] from arbor.local.toml
```

### `arbor.local.example.toml` (committed, workspace root)

```toml
[api]
ARBOR_PG_URL = "postgresql://arbor:arbor@localhost:5433/arbor_dev"
ARBOR_ORACLE_USER = "user"
ARBOR_ORACLE_PASSWORD = "password"
ARBOR_ORACLE_CONNECT_STRING = "localhost:1521/XEPDB1"
API_PORT = "3001"

[bff]
ARBOR_OIDC_ISSUER = "https://your-idp.example.com"
ARBOR_OIDC_CLIENT_ID = "your-client-id"
ARBOR_OIDC_CLIENT_SECRET = "your-client-secret"
ARBOR_OIDC_REDIRECT_URI = "http://localhost:3000/auth/callback"
ARBOR_SESSION_SECRET = "change-me-to-a-32-char-minimum-secret!"
ARBOR_API_URL = "http://localhost:3001"
BFF_PORT = "3000"

[bff.staging]
ARBOR_SESSION_SECRET = "change-me-staging-secret"
```

### Env composition for `arbor dev [mode]`

Precedence (later wins):

1. `os.environ` baseline
2. All keys from `[api]` in `arbor.local.toml`
3. All keys from `[bff]` in `arbor.local.toml`
4. All keys from `[bff.{bff_profile}]` if mode defines `bff_profile`
5. All keys from `[modes.{mode}]` in `arbor.toml` — mode vars always win

Then `subprocess.run(["turbo", "dev"], env=merged)`.

### `arbor run <section> -- <cmd>` — the universal env injector

A general-purpose command that loads a section from `arbor.local.toml`, merges it into the
environment, and execs the given command. This replaces `dotenv -e .env.local` everywhere,
including inside `apps/api/package.json`:

```json
"dev":        "../../arbor run api -- tsx watch src/index.ts",
"test":       "../../arbor run api -- vitest run",
"test:watch": "../../arbor run api -- vitest"
```

No `apps/api/.env.local` or `apps/bff/.env.local` needed at all — generated or otherwise.
`dotenv` and `dotenv-cli` are removed from `apps/api` devDeps.

`arbor setup` creates `arbor.local.toml` from `arbor.local.example.toml` (with a fresh
`ARBOR_SESSION_SECRET`), replacing the current per-app `.env.example` copy logic. No sync
step needed.

## Success Criteria

1. `arbor dev mock` (and `bff`, `oidc`, `staging`) launches the full stack with the same env as the corresponding `pnpm dev:*` script today.
2. `arbor db migrate` and `arbor db generate` work without `dotenv-cli`.
3. `arbor setup` produces `arbor.local.toml` from the example template with a freshly generated session secret.
4. Root `package.json` has one `dev` script; all `dev:*` scripts and `dotenv-cli` are gone.
5. `pnpm --filter @arbor/api test` works via `../../arbor run api -- vitest run`; no `.env.local` file needed.
6. No `dotenv` or `dotenv-cli` anywhere in the monorepo. No `python-dotenv` needed (TOML is stdlib ≥ 3.11).
