# PLAN.118 — Store DevTools Integration

## Context

Plan 110 describes `@arbor/router-devtools` as a standalone React overlay panel. With the
TCA router store (plans 113–117), the devtools have richer information available: every
action that passed through the reducer is logged (via `withLogging`), and the full
`RouterState` is a plain valtio-reactive object — no additional instrumentation needed.

This plan extends the devtools overlay to surface the TCA store's action history
alongside the route/loader/connection information from plan 110, and adds a store-level
time-travel capability (replay actions from history).

## Goal

1. **Action log panel** — when `RouterStoreProvider` is used with `debug={true}`, capture
   each `(action, prevState, nextState, effect | null)` tuple (mirroring `withLogging` but
   in-memory, not just console). Display as a scrollable timeline in the DevTools panel.

2. **Time-travel** — clicking an entry in the action log re-applies all actions up to that
   point against the initial state, resetting the store to that snapshot. This is
   implemented by replaying `store.send(action)` from a cloned initial state — no special
   reducer machinery required.

3. **State inspector** — rendered tree of the current `RouterState` snapshot (expandable
   JSON view); updates live on every action.

4. **Effect indicator** — each action in the log shows `⚡` (has effect) or `∅` (no
   effect), matching the `withLogging` format.

5. **Backward compatible with plan 110** — when using the hooks-based `RouterProvider`
   (no TCA), the action log and time-travel panels are hidden. Route inspector, loader
   status, and connection panels work regardless.

## API Surface

```typescript
// Dev-only; no-ops in production
<RouterDevtools
  position="bottom-right"
  defaultOpen={false}
  maxLogEntries={100}     // cap in-memory action log
  enableTimeTravel={true} // show time-travel controls
/>
```

## Implementation Notes

Action capture: wrap the reducer with a `withDevtools(reducer, collector)` middleware
(similar to `withLogging`) that pushes each action + state snapshot into a bounded
ring buffer stored in a separate valtio proxy. The DevTools panel subscribes to this
proxy directly — no React prop drilling.

Time-travel: replay is destructive (resets live store). Display a prominent confirmation
before replaying; disable in non-dev environments.

## Success Criteria

- Action log updates live with each navigation event when using `RouterStoreProvider`.
- Time-travel replays actions and the UI reflects the replayed state.
- Action log is absent (panel hidden) when using hooks-based `RouterProvider`.
- Zero production bundle impact — all devtools code is in `import.meta.env.DEV` guards.
