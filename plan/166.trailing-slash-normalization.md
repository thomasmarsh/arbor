# PLAN.166 — Trailing Slash Normalization

## Context

`parseSegments` already calls `.split('/').filter(Boolean)`, so `'users/:id/'` and
`'users/:id'` produce identical `Segment[]` output. URL splitting in `parse()` does the
same. `buildUrl` always emits paths without trailing slashes. Functionally there is no
requirement for trailing slashes — they are silently normalized.

However, the entire codebase (fixtures, tests, examples) uses trailing slashes by
convention, and the PBT generators in `segments.test.ts` always append `/` to each
segment. This creates the appearance of a hard requirement that doesn't exist. A developer
reading the code for the first time would reasonably conclude that `route(User, ':id')`
is invalid.

## Goal

Remove the apparent trailing-slash requirement by:

1. Adding explicit tests proving `'users/:id'` and `'users/:id/'` are equivalent at both
   parse time AND round-trip (parse → match → print).
2. Extending the PBT generators to test both trailing-slash and non-trailing-slash path
   forms so the invariants are proven for all segment combinations.
3. Updating `parseSegments`'s JSDoc to document that trailing slashes are normalized.
4. Optionally: drop trailing slashes from route definitions in `src/test-utils/fixtures.ts`
   and `examples/` to remove the convention entirely — do this only after the new tests
   pass (makes the normalization visible in the canonical examples).

## Change Surface & T-shirt Sizing

**XS** — no logic changes; tests and docs only.

Files:
- `src/core/segments.test.ts` — new cases + PBT generator update
- `src/core/segments.ts` — JSDoc only
- `src/test-utils/fixtures.ts` — optional: drop trailing slashes from definitions
- `examples/*.ts` — optional: drop trailing slashes from `route()` path args

## Dependencies

None. Independent of plans 163–165.

## Success Criteria

1. `it.each` table in `parseSegments` includes both `'users/:id/'` and `'users/:id'`
   asserting identical output.
2. A new `it.each` or `describe` block in `matchSegments` covers round-trip equivalence:
   `parseSegments(path)` and `parseSegments(path + '/')` produce the same match result
   against the same URL segments.
3. PBT `safePathArb` generates both trailing-slash and non-trailing-slash forms.
4. `pnpm --filter @arbor/router test` green.

## Notes

No changes to `parseSegments` logic — `filter(Boolean)` already handles it. This plan
is purely about making the invariant visible and tested so it cannot silently regress.

The `buildUrl` output format (no trailing slash) is intentional and stays as-is.
