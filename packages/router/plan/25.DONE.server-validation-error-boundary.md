# PLAN.25 — Server body validation and error boundary

## Context

`createServer` currently passes raw, unparsed request bodies directly to
handlers. `bodySchema` is stored in `_ctx` but never called. This means the
typed handler signature (`body: { email: string }`) is a lie at runtime — the
handler receives whatever the client sends. Two additional gaps:

- If a handler throws, the server crashes. No `try/catch` wraps handler
  invocation in `server.ts`.
- `decodeURIComponent()` is called on path segments
  (`define-routes.ts:108`) without a try/catch; a percent-encoded garbage
  segment throws a `URIError` that propagates up uncaught.

## Goal

Three minimal changes, all in `server.ts` and `walk.ts`:

1. **Body validation**: Before calling the handler, call
   `bodySchema.safeParse(await req.json())`. Return `400 Bad Request` on
   failure with the Zod error in the body. Skip validation (pass `{}` or
   `undefined`) if no `bodySchema` is defined for the route.

2. **Handler error boundary**: Wrap handler invocation in `try/catch`. If the
   handler throws, return `500 Internal Server Error` with a generic message
   (never leak the error message to the client).

3. **URL decode safety**: Wrap `decodeURIComponent` calls in a try/catch.
   If decoding fails, treat the segment as a non-match (return `null` from
   `matchSegments`) or return `400` from the server.

## Change Surface

- `src/server/server.ts`: body parsing + validation before handler call;
  try/catch around handler invocation.
- `src/core/walk.ts` or `src/core/segments.ts`: try/catch around
  `decodeURIComponent`.
- `src/server/server.test.ts`: add tests for:
  - malformed body → 400
  - handler throw → 500
  - percent-encoded garbage path → graceful 404 or 400 (not crash)

## Dependencies

- Plan 19 (phantom context split) should land first — `_ctx.bodySchema` must
  be accessible cast-free before this plan validates it.

## Success Criteria

- Sending a request with a body that fails `bodySchema` returns 400.
- A handler that throws returns 500; the server remains live.
- A request with `%zz` in the path does not crash the server.
- `npm test && npm run typecheck` clean.
