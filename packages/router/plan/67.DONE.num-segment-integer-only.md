# Plan 67 — Fix `num` segment to reject non-integers

## Context

The `num` (`#id`) and `opt-num` (`#id?`) segment kinds use `Number(url)` + `isNaN` to decide
if a path segment is numeric. `Number()` accepts inputs that are not integers:

| Input | `Number()` | `isNaN` result | Accepted today |
| ----- | ---------- | -------------- | -------------- |
| `'42'` | `42` | false | ✓ correct |
| `'1.5'` | `1.5` | false | ✓ wrong — passes |
| `'1e3'` | `1000` | false | ✓ wrong — passes |
| `'abc'` | `NaN` | true | ✗ correct |

The OpenAPI generator already emits `"type": "integer"` for `#id` params. The runtime
matcher should enforce the same contract: only bare integer strings like `'42'` or `'-5'`.

The fix: replace `Number(url)` + `isNaN` with `parseInt(url, 10)` + a round-trip string
check. If `String(parseInt(url, 10)) !== url` the input is not a bare integer and the match
should fail.

```ts
// before
const n = Number(url);
if (isNaN(n)) return null;

// after
const n = parseInt(url, 10);
if (isNaN(n) || String(n) !== url) return null;
```

Examples:
- `'42'` → `parseInt = 42`, `String(42) === '42'` → accepted ✓
- `'1.5'` → `parseInt = 1`, `String(1) !== '1.5'` → rejected ✓
- `'1e3'` → `parseInt = 1`, `String(1) !== '1e3'` → rejected ✓
- `-5` → `parseInt = -5`, `String(-5) === '-5'` → accepted ✓

The same fix applies to both `num` and `opt-num` branches in `matchSegments`.

## Goal

`#id` routes only match path segments that are bare integer strings. Non-integer numeric
strings (`'1.5'`, `'1e3'`) fall through to other routes or return 404, consistent with what
the OpenAPI output already promises.

## Change Surface

- `src/core/segments.ts` — two-line change in `matchSegments` (num + opt-num branches)
- `src/core/segments.test.ts` — add tests for `'1.5'`, `'1e3'`, `'-5'` in both `num` and `opt-num`

## Dependencies

None.

## Success Criteria

- `matchSegments` for `num` rejects `'1.5'` and `'1e3'`.
- `matchSegments` for `opt-num` rejects `'1.5'` and `'1e3'` (skips, same as non-numeric today).
- All existing tests pass.
- `pnpm test && pnpm typecheck` green.
