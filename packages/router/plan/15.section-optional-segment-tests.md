# PLAN.15 — Test multi-level sections and adjacent optional segments

## Problem

Two areas of the router have thin test coverage for edge cases:

**Multi-level sections**: `walkPrint` passes through section nodes without
unwrapping `child` from the route object. Existing tests only cover one level
of section nesting. Two nested sections (`section > section > route`) may not
print correctly — the child-drilling logic skips tagless nodes.

**Adjacent optional segments**: `matchSegments` silently skips non-matching
optional segments and leaves them in `rest`. With two consecutive optional
segments like `:name?/#id?/`, the behavior when only one is present is
untested and potentially ambiguous.

## Change

1. Add tests in `walk.test.ts` for nested sections (section inside section)
   for both `walkParse` and `walkPrint`.
2. Add tests in `segments.test.ts` for adjacent optional segments:
   - both present
   - first present, second absent
   - first absent, second present
   - neither present
3. Add tests in `define-routes.test.ts` for full roundtrip through nested
   sections.
4. Fix any bugs the tests reveal.

## Risk

Low. May uncover bugs — that's the point.
