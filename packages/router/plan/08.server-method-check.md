# PLAN.08 — Add HTTP method checking to `createServer.handle()`

## Problem

`createServer.handle()` parses the URL and dispatches to a handler by tag, but
never checks whether the incoming HTTP method matches the route's declared
method. A `DELETE` request to a `GET`-only endpoint will match and execute.

The `context.method` data already exists on every `httpRoute`/`openApiRoute`
node — it's just never consulted at dispatch time.

## Change

1. Accept `method` as a parameter in `handle()` (or via a request object).
2. After parsing, look up the matched route's declared method from the context
   map and compare it to the incoming method.
3. Return `{ status: 405, body: { error: "method not allowed" } }` on mismatch.
4. Add tests: correct method dispatches, wrong method returns 405.

## Notes

- This is a breaking change to the `handle()` signature — it currently takes
  `(url: URL, body?: unknown)` and would need a method argument.
- Consider whether `parse()` itself should be method-aware, or whether method
  checking belongs only in the server layer.

## Risk

Low. The data is already there; this is wiring it up.
