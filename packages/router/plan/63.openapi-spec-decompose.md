# Plan 63 — Decompose `walkSpec()` and fix `generateSpec` coupling

## Context

`generate-spec.ts` contains a 112-line `walkSpec()` function that handles six distinct concerns
in one body: wildcard filtering, path parameter extraction, query parameter inference, header
parameter assembly, operation metadata construction (operationId/summary/description/tags),
request body assembly, and response schema assembly. Two of those concerns (`isRequired()`
logic) are already duplicated inline — once for query params (line 70) and once for header
params (line 82).

Additionally, `generateSpec` is re-exported from `contexts/openapi-context.ts` line 64:

```ts
export { generateSpec } from '../openapi/generate-spec.js';
```

This is a separation-of-concerns violation. A file whose job is to define the `openApiRoute()`
factory and its types should not also re-export the spec generator. The re-export exists as a
convenience shortcut but creates an unexpected extra path to the same function.

## Goal

1. Extract helpers from `walkSpec()` so the orchestrator body is ~30 lines.
2. Eliminate the duplicated `isRequired()` inline logic.
3. Remove the `generateSpec` re-export from `openapi-context.ts` — the canonical path
   is `src/index.ts → openapi/index.ts → generate-spec.ts`.

## Change Surface

- `src/openapi/generate-spec.ts` — extract helpers, no behavior change
- `src/contexts/openapi-context.ts` — remove one re-export line
- `src/contexts/openapi-context.test.ts` — all `generateSpec` imports already come from
  the same file; the re-export removal does not affect the test file's own import style
  (it imports from `'../contexts/openapi-context.js'` — update to import from `'../openapi/generate-spec.js'`
  or leave via `src/index.ts` if the test is being updated holistically)

## Helpers to extract

| Helper | Lines currently inline | Responsibility |
|--------|------------------------|----------------|
| `isRequired(schema)` | 70, 82 | `!(s instanceof ZodOptional) && !(s instanceof ZodDefault)` |
| `buildPathParams(segments)` | 56–61 | path parameter objects |
| `buildQueryParams(shape, pathParamNames)` | 64–73 | query parameter objects |
| `buildHeaderParams(headerSchema)` | 75–86 | header parameter objects |
| `buildOperationMeta(ctx, tag, parameters)` | 90–99 | operationId/summary/etc |
| `buildRequestBody(bodySchema)` | 101–110 | requestBody object |
| `buildResponses(ctx)` | 112–135 | responses map |

`walkSpec()` becomes a loop over nodes that calls these helpers and assembles the result.

## Dependencies

None. This is a pure refactor inside one file.

## Success Criteria

- `pnpm test && pnpm typecheck && pnpm lint` pass with zero failures.
- `walkSpec()` body is ≤ 40 lines.
- `isRequired` is defined once.
- No `generateSpec` export in `openapi-context.ts`.
- `openapi-context.test.ts` imports `generateSpec` from `../openapi/generate-spec.js` (not via context).
