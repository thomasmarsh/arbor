# PLAN.38 — CORS / CSRF server wrapper (SEC1)

## Context

`spec.roadmap.md` Phase SEC1. Cross-origin and cross-site protections are
infra-level concerns that sit outside the routing loop. They need to run
before any route handler executes, and for OPTIONS preflight they need to
short-circuit the entire pipeline.

## Goal

A `withCors(server, config)` functional decorator (mirrors plan 37's
`withMetrics` shape) that handles CORS and CSRF at the server boundary.

### CORS

```typescript
interface CorsConfig {
  origins: string[] | '*';
  methods?: string[];
  allowedHeaders?: string[];
  credentials?: boolean;
  maxAge?: number;
}

const server = withCors(createServer(router, handlers), {
  origins: ['https://app.example.com'],
  credentials: true,
});
```

- For every request, add `Access-Control-Allow-Origin` (checked against the
  whitelist).
- For `OPTIONS` preflight, return `204 No Content` immediately without
  entering the route pipeline.
- Per-route overrides (plan 33) are respected — the decorator checks the
  matched route's `cors` field before applying the server-level policy.

### CSRF

For mutating methods (`POST`, `PUT`, `PATCH`, `DELETE`):
- Extract `x-csrf-token` from request headers.
- Compare against the `csrf-token` cookie value (HMAC-signed or plain).
- Mismatch → `403 Forbidden`.

CSRF validation is opt-in at the server level via `csrf: true` in the config.

## Change Surface

1. **New `src/server/with-cors.ts`**: Functional CORS/CSRF wrapper.
2. No changes to route contract or `createServer` — purely a server-output
   wrapper.
3. Tests:
   - Allowed origin → correct ACAO header.
   - Disallowed origin → no ACAO header or explicit rejection.
   - OPTIONS preflight → 204, no handler called.
   - CSRF mismatch (mutating method) → 403.
   - CSRF match → handler called normally.

## Dependencies

- Plan 33 (per-route CORS) should be designed first — this plan's
  implementation reads per-route `cors` overrides, so the contract field
  must exist.
- Plan 50 (enrichers public API) — use `withEnricher` as the mechanism instead
  of a pipeline entry. Each plan delivers a named enricher function.

Enricher name: `withCors`. Short-circuits with 200 on preflight or 403 on rejected origin.

## Success Criteria

- OPTIONS preflight never reaches a route handler.
- Valid origins get ACAO headers; invalid origins do not.
- CSRF mismatch on a mutating method returns 403.
- `npm test && npm run typecheck` clean.
