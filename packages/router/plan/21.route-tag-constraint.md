# PLAN.21 — Constrain `Route extends { tag: string }` in client and server

## Context

Issue 3 from plan 18: `createClient` and `createServer` both access
`route.tag` via unsafe casts (`route as Record<string, unknown>`) because their
`Route` generic parameter has no `{ tag: string }` bound. The actual union
always carries `tag`, but the compiler doesn't know.

## Goal

Pure type-level change — no runtime impact.

## Change Surface

1. **`fetch-client.ts`**: Change `Route` generic bound to
   `Route extends { tag: string }`. Replace the cast with direct `.tag` access.
2. **`server.ts`**: Same change.
3. Verify all call sites still compile (they pass `ChildUnion<C>` which always
   has `tag`).

## Success Criteria

- No `as Record<string, unknown>` cast for `.tag` access in either file.
- `npm test && npm run typecheck` clean.
- This is the smallest plan in the backlog — aim for under 10 lines changed.
