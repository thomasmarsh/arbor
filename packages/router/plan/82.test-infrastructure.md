# Plan 82 — Test infrastructure: fixture builders + table-driven refactor

## Context

Two recurring pain points across the test suite:

1. **Manual `RouteNode` construction in `walk.test.ts`**: ~80 lines of raw
   `{ _type: undefined, schema, path, segments, children }` objects are built inline
   and duplicated. Adding a new field to `RouteNode` (e.g., `segments` was added in
   Plan 59) silently breaks or requires touching every test fixture by hand.

2. **Repetitive single-assertion tests in `segments.test.ts`**: 15+ near-identical
   `it('parses X', () => expect(parseSegments('...')).toEqual([...]))` tests. The
   data-to-boilerplate ratio is inverted — the test file reads like boilerplate with
   data hidden inside it.

Both are code-quality smells that slow refactors and obscure intent. Neither fix
touches production code; both reduce maintenance burden before Wave 7 (segment
correctness) adds more cases.

## Goal

1. Create `src/test-utils/fixtures.ts` exporting canonical route tree builders
   used across test files.
2. Refactor `src/core/walk.test.ts` to build trees via `defineRoutes` + `route()`
   (the public API) rather than raw `RouteNode` objects.
3. Refactor `src/core/segments.test.ts` to use `it.each` tables for the parse and
   match test groups.

## Change Surface

- `src/test-utils/fixtures.ts` — new file; not exported from `src/index.ts`
- `src/core/segments.test.ts` — convert repetitive `it` blocks to `it.each`
- `src/core/walk.test.ts` — replace raw node construction with `routeFixtures.*`

## Fixture design

```typescript
// src/test-utils/fixtures.ts
import z from 'zod';
import { route, section, defineRoutes } from '../core/define-routes.js';

const Users    = z.object({ tag: z.literal('users') });
const User     = z.object({ tag: z.literal('user'),    id: z.string() });
const Settings = z.object({ tag: z.literal('settings') });
const Org      = z.object({ tag: z.literal('org'),     orgId: z.string() });
const Project  = z.object({ tag: z.literal('project'), projectId: z.number() });
const Issue    = z.object({ tag: z.literal('issue'),   issueId: z.string() });

export const routeFixtures = {
  // users/ → :id/ → settings/
  userTree: () =>
    defineRoutes([
      route(Users, 'users/', [
        route(User, ':id/', [route(Settings, 'settings/')]),
      ]),
    ]),

  // orgs/:orgId/ → #projectId/ → :issueId/
  orgTree: () =>
    defineRoutes([
      route(Org, 'orgs/:orgId/', [
        route(Project, '#projectId/', [route(Issue, ':issueId/')]),
      ]),
    ]),

  // Combined tree used in multi-root walk tests
  combinedTree: () =>
    defineRoutes([
      route(Users, 'users/', [
        route(User, ':id/', [route(Settings, 'settings/')]),
      ]),
      route(Org, 'orgs/:orgId/', [
        route(Project, '#projectId/', [route(Issue, ':issueId/')]),
      ]),
    ]),
};
```

Fixtures use the public `defineRoutes` + `route` API. They get `segments` computed
automatically (no manual `parseSegments` calls). If `RouteNode` grows a new field,
fixture construction remains valid with no edits.

## `it.each` refactor — segments.test.ts

Before (×15 near-identical tests):

```typescript
it('parses a single literal', () => {
  expect(parseSegments('users/')).toEqual([{ kind: 'lit', value: 'users' }]);
});
it('parses a string param', () => {
  expect(parseSegments(':id/')).toEqual([{ kind: 'str', name: 'id' }]);
});
// …
```

After (1 table):

```typescript
it.each([
  ['single literal',  'users/',    [{ kind: 'lit', value: 'users' }]],
  ['string param',    ':id/',      [{ kind: 'str', name: 'id' }]],
  ['number param',    '#id/',      [{ kind: 'num', name: 'id' }]],
  ['optional string', ':id?/',     [{ kind: 'opt-str', name: 'id' }]],
  ['optional number', '#id?/',     [{ kind: 'opt-num', name: 'id' }]],
  ['wildcard',        '*rest/',    [{ kind: 'wildcard', name: 'rest' }]],
] satisfies [string, string, Segment[]][])(
  'parseSegments: %s',
  (_, input, expected) => expect(parseSegments(input)).toEqual(expected),
);
```

Same coverage, ~60% fewer lines.

## Dependencies

None. Pure test-layer refactor; no production code changes.
