# PLAN.19 ‚Äî Split phantom context from runtime context (`_ctx` field)

## Context

Issue 1 from plan 18: `context` is simultaneously a phantom type carrier and a
concrete runtime data bag. The field names on the type-level interface differ
from those on the runtime object (`body` vs `bodySchema`, `response` vs
`responseSchemas`). This forces `as unknown as HttpContext<...>` casts in every
read site.

## Goal

Introduce a concrete `_ctx` field that holds the runtime data. Keep `context`
as a pure phantom (typed `undefined as never`, exactly like `_type`/`_child`).

## Change Surface

1. **`route-node.ts`**: Add `_ctx?: unknown` to `RouteNode`. Mark it as
   concrete (not phantom).
2. **`http-context.ts`**: Store runtime data `{ method, bodySchema?, responseSchemas }`
   into `_ctx` instead of `context`. Remove the `as unknown as` cast.
3. **`openapi-context.ts`**: Same ‚Äî store `{ responseSchemas }` into `_ctx`.
4. **`walk.ts`**: Read `node._ctx` instead of `node.context as {...}`.
5. **`fetch-client.ts`**: Read `node._ctx` instead of `node.context as {...}`.
6. **`define-routes.ts`**: Read `node._ctx.method` instead of
   `(node.context as {...}).method`.
7. **`openapi-context.test.ts`**: Update assertion from
   `(r.context as unknown as {...}).responseSchemas` ‚Üí `r._ctx.responseSchemas`.

## Success Criteria

- Zero `as unknown as` casts in non-test code (excluding the accepted list in
  plan 18).
- All existing tests pass unchanged except the one test cast in 1b.
- `npm test && npm run typecheck` clean.
