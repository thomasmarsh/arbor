# PLAN.56 — Tech debt: core/define-routes.ts coupled to HTTP context

## Context

`core/define-routes.ts` currently imports `getHttpMeta` and `CorsConfig` from
`contexts/http-context.ts` and contains six HTTP-specific collector functions:

```
collectMethods, collectBodySchemas, collectRequestHeaderSchemas,
collectResponseHeaderSchemas, collectRateLimits, collectCors
```

These all call `getHttpMeta(node)` to extract fields from the route's `_meta`
bag, then return a flat `Record<string, T>` keyed by tag. The resulting maps
(`methodMap`, `bodySchemaMap`, `headerSchemaMap`, `responseHeaderSchemaMap`,
`rateLimitMap`, `corsMap`) are attached directly to the router object returned
by `defineRoutes()`.

This violates the stated architectural constraint:

> The core types like RouteNode should be as domain independent as much as
> possible.

`core/` must not know about HTTP methods, rate limits, CORS, or any other
context-specific field. A future context (e.g. WebSocket, gRPC) should not
require changes to `core/define-routes.ts`.

## Goal

Make `defineRoutes()` context-agnostic. The HTTP-specific collector functions
and their output maps should live in the HTTP context layer, not in `core/`.

## Proposed Approach

### 1. Add a generic walk-collector to `core/`

Introduce a small utility in `core/walk.ts` (or a new `core/collect.ts`):

```typescript
export function walkCollect<T>(
  nodes: WalkNode[],
  extractor: (node: WalkNode, tag: string) => T | undefined,
): Record<string, T> {
  const map: Record<string, T> = {};
  for (const node of nodes) {
    if (node.schema !== null) {
      const tag = getTag(node.schema);
      if (tag) {
        const value = extractor(node, tag);
        if (value !== undefined) map[tag] = value;
      }
    }
    if (node.children.length > 0) {
      Object.assign(map, walkCollect(node.children as WalkNode[], extractor));
    }
  }
  return map;
}
```

### 2. Strip HTTP-specific collectors from `define-routes.ts`

Remove `collectMethods`, `collectBodySchemas`, `collectRequestHeaderSchemas`,
`collectResponseHeaderSchemas`, `collectRateLimits`, `collectCors`, and their
output keys from the router object. Remove the `getHttpMeta` / `CorsConfig`
imports from `core/`.

The base router returned by `defineRoutes()` exposes only:
`_type`, `_ctxMap`, `children`, `parse`, `parseDiagnostics`, `print`.

### 3. Move HTTP map construction to the HTTP context layer

Add a new function (e.g. `collectHttpMaps(nodes)` in `contexts/http-context.ts`
or a new `contexts/http-define.ts`) that uses `walkCollect` to build all the
HTTP maps:

```typescript
export function collectHttpMaps(nodes: WalkNode[]) {
  return {
    methodMap:             walkCollect(nodes, (n) => getHttpMeta(n)?.method),
    bodySchemaMap:         walkCollect(nodes, (n) => getHttpMeta(n)?.bodySchema),
    headerSchemaMap:       walkCollect(nodes, (n) => getHttpMeta(n)?.headerSchema),
    responseHeaderSchemaMap: walkCollect(nodes, (n) => getHttpMeta(n)?.responseHeaderSchemas),
    rateLimitMap:          walkCollect(nodes, (n) => getHttpMeta(n)?.rateLimit),
    corsMap:               walkCollect(nodes, (n) => getHttpMeta(n)?.cors),
  };
}
```

### 4. Compose in `defineRoutes()` via a plugin/options argument (preferred)

Rather than a separate `collectHttpMaps` call at the call site, keep the
ergonomic single `defineRoutes()` call by accepting an optional `collect`
options bag:

```typescript
defineRoutes(children, { collect: collectHttpMaps })
```

`defineRoutes` calls `collect(nodes)` and spreads the result onto the returned
router. This keeps the call site unchanged for HTTP routes while making
`defineRoutes` itself context-free — it does not know what `collectHttpMaps`
contains.

Alternatively, expose `nodes` on the returned router so the HTTP layer can run
collectors after the fact without changing `defineRoutes`'s signature at all.

## Change Surface

1. **`core/walk.ts`** (or new `core/collect.ts`): Add `walkCollect`.
2. **`core/define-routes.ts`**: Remove all HTTP-specific collectors and imports.
   Decide and implement the plugin/options strategy.
3. **`contexts/http-context.ts`** (or new `contexts/http-define.ts`): Add
   `collectHttpMaps` using `walkCollect`.
4. **`server/server.ts`**: If the HTTP maps move off the router type, update
   `createServer`'s parameter type accordingly.
5. **Tests**: No behavior change; verification chain must stay green.

## Dependencies

- None blocking. Can be worked independently of any open feature plan.

## Success Criteria

- `core/define-routes.ts` has zero imports from `contexts/`.
- `core/` has no knowledge of HTTP methods, rate limits, CORS, or any
  `HttpContextData` field.
- `defineRoutes()` call sites and `createServer()` call sites are unchanged
  (or the change is purely additive).
- `pnpm test && pnpm run typecheck && pnpm run lint` clean.
