# Plan 61 — Type the _meta bag

## Context

`RouteNode` has a fifth type parameter `Meta = Record<string, unknown>`. At runtime,
`_meta` holds the plugin metadata set by context-specific route builders:

- `httpRoute()` stores `HttpContextData` (method, bodySchema, headerSchemas, responseSchemas, rateLimit, cors, cookies)
- `openApiRoute()` stores `OpenApiCtxData` (summary, tags, operationId, etc.)

Both use the same `_meta` slot but the type is `Record<string, unknown>` everywhere.
Every consumer casts it to its expected shape:

```typescript
// in walk.ts
const meta = node._meta as { querySchema?: z.ZodObject<any, any> };

// in http-context.ts
export function getHttpMeta(node: WalkNode): HttpContextData | undefined {
  return node._meta as HttpContextData | undefined;
}

// in openapi-context.ts
export function getOpenApiMeta(node: WalkNode): OpenApiCtxData | undefined {
  return node._meta as OpenApiCtxData | undefined;
}
```

There is no compile-time guarantee that `_meta` actually contains what each accessor
expects. Adding a field to `HttpContextData` does not produce an error if a call site
forgets to set it. The contract is distributed across multiple files with no central
enforcement.

## Goal

Make `_meta` carry its concrete type through the `RouteNode` generic parameter so that
`getHttpMeta` returns a properly typed value without unsafe casts, and incorrect `_meta`
shapes produce compile errors at the point of construction rather than silent runtime
misbehavior.

## Proposed Approach

`Meta` is already the fifth type parameter of `RouteNode`. The issue is that the
helper functions (`getHttpMeta`, `getOpenApiMeta`) operate on `WalkNode` which aliases
`RouteNode<unknown, any, any, any, any>` — discarding the `Meta` type.

### Option A — Narrow WalkNode to carry Meta (recommended)

Introduce typed variants of `WalkNode` for each context layer:

```typescript
type HttpWalkNode = RouteNode<unknown, any, any, any, HttpContextData>;
type OpenApiWalkNode = RouteNode<unknown, any, any, any, OpenApiCtxData>;
```

Update `getHttpMeta`:

```typescript
export function getHttpMeta(node: RouteNode<unknown, any, any, any, HttpContextData>): HttpContextData {
  return node._meta!;  // no cast needed — Meta is HttpContextData
}
```

`walkParse` and `walkCollect` work with `WalkNode` (untyped Meta) for core operations.
HTTP-specific collectors accept `HttpWalkNode[]` and get typed `_meta` access.

**Pros:** Eliminates all casts in accessors; type-safe at construction sites.  
**Cons:** Requires narrowing `WalkNode` at the point where HTTP-specific collectors
are called (one cast at the boundary, not scattered).

### Option B — Brand tag on Meta (alternative)

Add a `readonly _kind: 'http' | 'openapi'` discriminant to `HttpContextData` and
`OpenApiCtxData`. `getHttpMeta` performs a runtime check and returns typed value.

**Pros:** Works with the untyped `WalkNode` everywhere without generic narrowing.  
**Cons:** Adds a runtime discriminant just to satisfy the type system — the `_kind`
field carries no semantic meaning.

### Recommendation

Option A. The single cast at the collector boundary is better than scattered unsafe
casts or a synthetic runtime discriminant. The goal is eliminating casts in the
accessors, not eliminating all casts everywhere.

## Change Surface

| File | Change |
|------|--------|
| `core/walk.ts` | Export `HttpWalkNode` alias (or define in http-context) |
| `contexts/http-context.ts` | Update `getHttpMeta` signature; remove `as HttpContextData` cast |
| `contexts/openapi-context.ts` | Update `getOpenApiMeta` signature; remove `as OpenApiCtxData` cast |
| `core/walk.ts` | Remove `node._meta as { querySchema?: ... }` cast; use `getHttpMeta` |
| `core/route-node.ts` | No change to RouteNode shape needed |

## Dependencies

- Can run in parallel with Plan 59 (walk decompose) — both touch `walk.ts` but at
  different lines.
- Plan 60 (server decompose) should land after Plan 61 so server code benefits from
  typed `_meta` access.

## Success Criteria

- `getHttpMeta` and `getOpenApiMeta` contain no `as` casts.
- `walk.ts` contains no inline `_meta as { ... }` casts.
- `httpRoute()` construction sites produce a compile error if `HttpContextData`
  is missing a required field.
- `pnpm test && pnpm typecheck && pnpm lint` clean.
- No behavior change.
