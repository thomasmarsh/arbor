# Plan 64 — Fix cast-and-mutate in `openApiRoute()`

## Context

`openapi-context.ts` lines 57–61:

```ts
const node = httpRoute(schema, method, path, { ..., response: options.response }, children);
if (options.meta) {
  (node._meta as OpenApiCtxData).meta = options.meta;
}
return node as RouteNode<z.infer<S>, [...C], OpenApiContext<...>, never, OpenApiCtxData>;
```

`httpRoute()` returns a node whose `_meta` is typed as `HttpContextData`. The call to
`openApiRoute()` then casts `node._meta` to `OpenApiCtxData` and mutates the `.meta` field
through the cast. Two problems:

1. **Mutation through a cast** — TypeScript sees the write as safe, but there is no actual
   runtime guarantee the object was initialized with a `meta` field slot. If `httpRoute`
   ever returns a sealed or frozen object the write silently fails.

2. **Forced return cast** — `return node as RouteNode<..., OpenApiCtxData>` coerces the type
   even though the runtime object's `_meta` is typed as `HttpContextData`. The cast
   papers over the fact that the node was constructed without `meta`.

The fix is to build the `_meta` object with all fields — including `meta` — at construction
time rather than patching it after the fact. Because `openApiRoute` has access to all options
before calling `httpRoute`, it can pass a fully-formed `HttpContextData & { meta?: OpenApiMeta }`
object.

## Goal

Eliminate the post-construction cast-and-mutate. The `_meta` of the returned node must be
structurally correct before the return statement, without any cast.

## Approach

Option A (preferred): construct `_meta` directly in `openApiRoute()` without calling `httpRoute`,
duplicating the small metadata construction block. This removes the dependency and the cast.

Option B: pass an `_metaExtra` option through `httpRoute`'s options so it can be merged into
`_meta` at construction time. Avoids duplication but couples `httpRoute` to an extension slot.

Option A is preferred because `openApiRoute` is a thin wrapper; duplicating the ~15-line
`_meta` construction is a better tradeoff than adding an escape hatch to `httpRoute`.

None. This is isolated to `openapi-context.ts`.
