# Plan 83 — Inline snapshots + diagnostics testing layer

## Context

Two snapshot gaps in the current test suite:

1. **External snapshot file**: `src/openapi/__snapshots__/generate-spec.test.ts.snap`
   is the only snapshot test. External `.snap` files are invisible during PR review —
   a structural regression in the OpenAPI output goes undetected until someone manually
   opens the snapshot. Vitest supports `toMatchInlineSnapshot()` which keeps the
   expected value co-located with the test.

2. **`ParseDiag` is untested structurally**: `ParseDiag` is a rich discriminated union
   (`no-match | ambiguous | schema-mismatch | …`) but no test asserts what the error
   object *looks like*. A change to the diagnostic shape is entirely invisible in CI
   unless a field name happens to be referenced in a downstream test.

## Goal

1. Convert `generate-spec.test.ts` to `toMatchInlineSnapshot()` and delete the
   external `.snap` file.
2. Add a `ParseDiag` snapshot suite in `src/core/walk.test.ts` (or a new
   `src/core/diagnostics.test.ts`) that snapshots the error shape for each major
   failure kind.
3. Add inline snapshots for server error-path responses in `src/server/server.test.ts`
   for cases where the exact body shape matters.

## Change Surface

- `src/openapi/generate-spec.test.ts` — replace `toMatchSnapshot()` with
  `toMatchInlineSnapshot()` (run `pnpm test -u` to populate)
- `src/openapi/__snapshots__/` — delete after migration
- `src/core/walk.test.ts` (or `src/core/diagnostics.test.ts`) — add ParseDiag
  snapshot group
- `src/server/server.test.ts` — add inline snapshots for 400/404/405/500 body shapes

## Snapshot rules enforced here

Per `plan/testing.md` §4:

- Inline only; no new external snapshot files.
- Each snapshot ≤ 30 lines.
- Snapshot the *structure*, not prose messages.

## ParseDiag snapshot targets

At minimum, one test per major ParseDiag kind:

```typescript
describe('ParseDiag shapes', () => {
  it('no-match diagnostic', () => {
    const r = defineRoutes([route(Home, 'home/')]);
    expect(r.parse(new URL('https://h/unknown')).getErr())
      .toMatchInlineSnapshot(`...`);
  });

  it('schema-mismatch diagnostic — number param given non-number', () => {
    const r = defineRoutes([route(Item, 'items/#id/')]);
    expect(r.parse(new URL('https://h/items/abc/')).getErr())
      .toMatchInlineSnapshot(`...`);
  });
});
```

The exact snapshot values are populated on first run via `pnpm test -u` and then
committed. They become a regression guard — any accidental rename of a `ParseDiag`
field causes an immediate test failure with a clear diff.

## Server error body snapshots

The `validateInput`, `resolveHandler`, and `validateResponse` error returns each have
a specific `{ error: string }` shape. Inline snapshots prevent silent wording drift:

```typescript
it('405 body shape', () => {
  expect(resolveHandler({}, 'any', 'POST', 'GET')).toMatchInlineSnapshot(`
    {
      "body": {
        "error": "method not allowed",
      },
      "ok": false,
      "status": 405,
    }
  `);
});
```

## Dependencies

None. Pure test additions; no production code changes.
