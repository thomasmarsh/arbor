# Plan 52 — Spike: openApiRoute wraps httpRoute

## Context

`OpenApiContext` already extends `HttpContext` (openapi-context.ts:21), declaring that OpenAPI routes are a superset of HTTP routes. But `openApiRoute` reimplements `_ctx` construction in parallel with `httpRoute`, writing `method`, `bodySchema`, and `responseSchemas` a second time.

This duplication is the blocker for plan 53 (removing domain fields from `RouteCtx`). Until there is one canonical `_ctx` shape (http's), the cleanup can't proceed cleanly.

This spike validates whether `openApiRoute` can be implemented as a thin wrapper over `httpRoute` with no behavior change.

## Goal

Rewrite `openApiRoute` to call `httpRoute` internally, then append `meta` to the returned `_ctx`. All existing tests must pass unchanged.

## Change Surface

Single file: `src/contexts/openapi-context.ts`.

Current `openApiRoute` body (lines 46–59) constructs a `RouteNode` from scratch. Replace with:

```typescript
export function openApiRoute<...>(
  schema, method, path, options, children,
) {
  const node = httpRoute(schema, method, path, { body: options.body, response: options.response }, children);
  if (options.meta) {
    (node._ctx as Record<string, unknown>).meta = options.meta;
  }
  return node as RouteNode<z.infer<S>, [...C], OpenApiContext<Method, Body, InferResponseMap<Res>>>;
}
```

The cast on `node._ctx` is unavoidable here because `_ctx` is typed as `RouteCtx` (which already has `meta?: Record<string, unknown>`) so this is a narrowing cast, not an escape.

Note: `openApiRoute` currently does NOT support `querySchema`, `headerSchema`, `responseHeaderSchemas`, or `rateLimit` — those come from `httpRoute`'s option bag and are simply not passed here. That is intentional: OpenAPI routes that need those features should use `httpRoute` directly or extend the `openApiRoute` options object. Do not add those options in this spike — just verify the existing behavior is preserved.

## Dependencies

- None. This is a self-contained refactor within one file.
- Must complete before plan 53 (RouteCtx cleanup).

## Success Criteria

- `pnpm test && pnpm typecheck && pnpm lint` all pass
- No changes to any test file
- `generate-spec.ts` (which reads `_ctx.meta`) still works correctly
- The `openApiRoute` return type is unchanged
