# PLAN.32 — Cookie handling in the contract

## Context

Session cookies, CSRF tokens, and preference cookies are a routine part of
HTTP APIs. Right now they are entirely invisible to the contract — handlers
must reach into `ctx.req` and parse them manually with no type safety or
OpenAPI documentation.

ts-rest can declare cookies as part of the route contract, making them typed
inputs to the handler and documented outputs in the spec.

This plan is **blocked on Q10** (WinterCG CookieStore vs. custom abstraction)
and on the typed-request-headers design (plan 30), which establishes the
pattern for validated contract inputs beyond params/body/query.

## Goal

Allow a route to declare an optional `cookies` Zod schema. Validated cookie
values flow into `ctx` alongside headers, params, body, and query. Response
cookie-setting is expressed as a typed `cookies` field in the handler return.

```typescript
httpRoute({
  method: 'POST',
  path: 'session/',
  cookies: z.object({
    'csrf-token': z.string(),
  }),
  responses: {
    200: {
      body: SessionSchema,
      cookies: z.object({ 'session-id': z.string() }),
    },
  },
});
```

## Change Surface

Depends on Q10 resolution. If WinterCG `CookieStore` is available:
- Parse via the `CookieStore` API.

If custom abstraction:
- Parse `Cookie` header string directly using a minimal utility.

Either way:
1. **`http-context.ts`**: Add optional `cookies` schema to the route contract.
2. **`server.ts`**: Extract, validate, and merge cookies into `ctx`. On
   validation failure, return 400.
3. **Handler return type**: Optional `cookies` map → written as `Set-Cookie`
   headers on the response.
4. Tests: missing required cookie → 400; valid cookies → typed in ctx; response
   cookies set on `Response` headers.

## Dependencies

- **Q10 must be resolved** before implementation begins.
- Plan 30 (typed request headers) — establishes the validated-contract-input
  pattern; cookies follow the same structure.
- Plan 29 (typed response headers) — response cookies extend the response
  header shape.

## Success Criteria

- Missing required cookie in the contract returns 400.
- Validated cookies are available typed in the handler ctx.
- Response cookies declared in the contract are set as `Set-Cookie` headers.
- `npm test && npm run typecheck` clean.
