# Plan 68 — Wildcard captures `string` not `string[]`

## Context

`matchSegments` captures a wildcard (`*rest`) as `string[]`:

```ts
case 'wildcard':
  next[seg.name] = urlSegments.slice(urlIndex);
  urlIndex = urlSegments.length;
  break;
```

Every caller that uses the captured value must manually `.join('/')` to get a path string.
More critically, `buildUrl` currently does not special-case wildcard:

```ts
seg.kind === 'lit' ? seg.value : encodeURIComponent(String(allParams[seg.name]))
```

`String(['a', 'b', 'c'])` → `'a,b,c'` — comma-joined, not slash-joined. This means
`print()` does not round-trip routes with wildcard params correctly today.

The common mental model for `*rest` is "the remaining path as a string"
(`'images/avatar.png'`), not an array. An array is only useful if you want to iterate the
segments individually, which is a secondary concern that can be done with `.split('/')` on
the string.

## Goal

Change wildcard to capture a joined `string`. Fix `buildUrl` to emit the value without
percent-encoding the slashes. Make `print()` round-trip correctly for wildcard routes.

## Approach

**`matchSegments`**: join rather than slice.

```ts
case 'wildcard':
  next[seg.name] = urlSegments.slice(urlIndex).join('/');
  urlIndex = urlSegments.length;
  break;
```

**`buildUrl`**: special-case wildcard so the `/`-separated value is not percent-encoded.

```ts
.map((seg) => {
  if (seg.kind === 'lit') return seg.value;
  if (seg.kind === 'wildcard') return String(allParams[seg.name]); // slashes must not be encoded
  return encodeURIComponent(String(allParams[seg.name]));
})
```

**Route schemas using wildcard**: users who declared the wildcard param field as
`z.array(z.string())` must change it to `z.string()`. This is a **breaking change** to
user-facing Zod schemas. The examples and tests in this repo need to be updated; no
migration path is needed given we are pre-1.0.

## Change Surface

- `src/core/segments.ts` — one-line change in `matchSegments`
- `src/core/walk.ts` — `buildUrl` wildcard case
- `src/core/segments.test.ts` — update wildcard capture assertions from `string[]` to `string`
- `src/core/walk.test.ts` — update any wildcard-param `buildUrl` tests
- Any example or test that references a wildcard param as an array

## Dependencies

None. Isolated to `segments.ts` and `walk.ts`.

## Success Criteria

- `matchSegments` for `*rest` on `['a', 'b', 'c']` returns `{ rest: 'a/b/c' }`.
- `matchSegments` for `*rest` on `[]` returns `{ rest: '' }`.
- `buildUrl` with a wildcard param correctly reconstructs the path (no comma join, no
  percent-encoded slashes).
- `print(parse(url).getOrThrow())` round-trips for routes containing wildcards.
- `pnpm test && pnpm typecheck` green.
