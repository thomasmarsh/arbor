# Plan 65 — Module barrel cleanup and public export surface audit

## Context

Three module-structure problems are present:

### 1. Empty barrel files

`src/contexts/index.ts` — 0 bytes.  
`src/security/index.ts` — 0 bytes.

These files exist but export nothing. They are probably artifacts of initial scaffolding.
Nothing imports from either path (confirmed via `rg`). They add noise without benefit.

### 2. `generateSpec` exported via wrong module (duplicate path)

`src/contexts/openapi-context.ts` line 64 re-exports `generateSpec`:

```ts
export { generateSpec } from '../openapi/generate-spec.js';
```

The canonical public path is `src/index.ts → src/openapi/index.ts → generate-spec.ts`.
The re-export from `openapi-context.ts` creates a second public path for the same function
and blurs the module boundary: a file whose job is route definition shouldn't also be a
conduit to spec generation. (This re-export is removed as part of Plan 63.)

### 3. Public API gaps in root `src/index.ts`

| Symbol | Where defined | In root index.ts? |
|--------|--------------|-------------------|
| `ParseDiag` | `core/walk.ts`, re-exported from `core/index.ts` | **No** |
| `HandlerCtx` | `server/server.ts`, in `server/index.ts` | **No** |
| `WalkNode` | `core/walk.ts` | No (internal — not a gap) |

Users who build typed handler maps or work with diagnostics need `HandlerCtx` and `ParseDiag`
from the package root. Currently they must import from deep paths (`../../server/index.js`
or `../../core/walk.js`), which breaks when the package is published.

## Goal

1. Delete `src/contexts/index.ts` and `src/security/index.ts`.
2. Add `ParseDiag` and `HandlerCtx` to root `src/index.ts`.
3. Remove the spurious `generateSpec` re-export from `openapi-context.ts` (coordinate with Plan 63).

## Change Surface

- `src/contexts/index.ts` — delete
- `src/security/index.ts` — delete
- `src/index.ts` — add two type exports
- `src/contexts/openapi-context.ts` — remove one line (coordinate with Plan 63)

## Dependencies

- The `generateSpec` re-export removal overlaps with Plan 63. Whichever plan runs first owns
  that change; the other plan's success criteria should note the removal is already done.

## Success Criteria

- `src/contexts/index.ts` and `src/security/index.ts` do not exist.
- `ParseDiag` and `HandlerCtx` importable from the package root.
- `pnpm test && pnpm typecheck && pnpm lint` pass.
- No `import ... from '*/contexts/index'` or `'*/security/index'` anywhere in the repo.
