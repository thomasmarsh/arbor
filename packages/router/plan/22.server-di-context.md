# PLAN.22 — Contextual Dependency Injection in `createServer`

## Context

`spec.enhancements.md` Phase 2, item 1. Right now handlers receive individual
arguments loosely. The goal is a unified `ctx` argument containing validated
path params, query params, and request body — all narrowed to the specific
route's schema types.

## Goal

```typescript
// Before (approximate current shape)
handler: (params, body, query) => Response

// After
handler: (ctx: { params: Params; body: Body; query: Query; req: Request }) => Response
```

All three schema sources (path schema, body schema, query schema) feed a single
typed `ctx` object. The handler cannot access raw params without going through
the validated bag.

## Change Surface

1. **`http-context.ts`**: Update `HttpContext` to include `query` alongside
   `body`/`response`.
2. **`server.ts`**: Merge validated path params + query params + body into a
   single `ctx` object before calling the handler.
3. **`ServerHandlers` mapped type**: Update the handler signature to accept
   `ctx` rather than separate arguments.
4. Tests: update `server.test.ts` to use the new signature.

## Dependencies

- Plan 19 (phantom context split) should land first — the `_ctx` field makes
  reading `bodySchema` and `querySchema` at runtime safe and cast-free.

## Success Criteria

- Handlers receive a single typed `ctx` object.
- `npm test && npm run typecheck` clean.
