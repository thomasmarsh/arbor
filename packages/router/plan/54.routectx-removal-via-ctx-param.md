# Plan 54 — Remove RouteCtx via a `Ctx` Type Parameter on RouteNode

## Context

Plan 53 introduced `getHttpCtx` / `getOpenApiCtx` typed accessors and moved all
consumer code off direct `_ctx` field access. However, `RouteCtx` remains in
`src/core/route-node.ts` because removing it and using `Record<string, unknown>`
as the `_ctx` field type breaks test files under `noPropertyAccessFromIndexSignature`.

The test files access `r._ctx?.querySchema`, `r._ctx?.method`, etc. with dot
notation. With `Record<string, unknown>`, TypeScript TS4111 requires bracket
notation and the tests cannot be modified without touching them.

The clean solution is to add a `Ctx` type parameter to `RouteNode` so that
`httpRoute()` returns `RouteNode<..., HttpContextData>` — giving `_ctx` an
explicitly typed shape per factory, enabling dot-notation access in tests without
leaking HTTP field names into `route-node.ts`.

## Goal

- `RouteCtx` interface removed from `route-node.ts`
- `_ctx` field typed as `Ctx` (a new 5th type parameter defaulting to `Record<string, unknown>`)
- `httpRoute()` returns `RouteNode<..., HttpContextData>`, preserving test access
- `openApiRoute()` returns `RouteNode<..., OpenApiCtxData>`
- `pnpm test && pnpm run typecheck && pnpm run lint` all pass

## Change Surface

### 1. `src/core/route-node.ts`

Add 5th type parameter `Ctx = Record<string, unknown>`:

```typescript
export interface RouteNode<
  R,
  C extends RouteNode<unknown, any, any, any, any>[] = [],
  Context = never,
  SectionParams extends string = never,
  Ctx = Record<string, unknown>,
> {
  // ...
  _ctx?: Ctx;
}
```

Remove `RouteCtx` interface. Update the 4-arg `RouteNode<unknown, any, any, any>`
shorthand throughout (add a 5th `any`).

### 2. Cascade `RouteNode<unknown, any, any, any>` → `RouteNode<unknown, any, any, any, any>`

Files that use the 4-arg shorthand for generic contexts:
- `src/core/route-node.ts` (`FlattenChildrenImpl`, `ChildUnion`, `Derive`, `CtxMap`, etc.)
- `src/core/walk.ts` (`WalkNode`, `FlattenChildrenImpl` constraints)
- `src/core/define-routes.ts`
- `src/client/fetch-client.ts`
- `src/server/server.ts`
- `src/openapi/generate-spec.ts`
- `src/contexts/http-context.ts`, `openapi-context.ts`

### 3. `src/contexts/http-context.ts`

`httpRoute()` return type includes `HttpContextData` as the 5th `RouteNode` parameter:

```typescript
): RouteNode<
  ...,
  HttpContextData
>
```

### 4. `src/contexts/openapi-context.ts`

`openApiRoute()` return type uses `OpenApiCtxData` as the 5th parameter.

### 5. `getHttpCtx` / `getOpenApiCtx` signatures

Update parameter type to `{ _ctx?: Ctx }` where appropriate (or keep the
`Record<string, unknown>` widened form — callers pass nodes whose `Ctx` is
assignable to `Record<string, unknown>`).
