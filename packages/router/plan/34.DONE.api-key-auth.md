# PLAN.34 — API key authentication strategy

## Context

Plan 39 (SEC2) handles JWT/bearer token auth via a `protectedRoute` factory.
API key authentication is a distinct and common pattern — keys arrive via
a header (`x-api-key`) or query parameter, are validated against a store
rather than a signature, and carry different identity/permissions semantics
than session tokens.

Treating it as a second auth strategy ensures the two can be composed and
documented correctly in the OpenAPI spec (`securitySchemes`).

## Goal

A `apiKeyRoute(node, { via: 'header' | 'query', name: string })` higher-order
factory that mirrors the `protectedRoute` pattern from plan 39:

```typescript
const route = apiKeyRoute(
  httpRoute({ method: 'GET', path: 'data/', ... }),
  { via: 'header', name: 'x-api-key' },
);

// Handler ctx gains a mandatory resolved key identity
handler: async ({ params, apiKey }) => { ... }
```

The server reads the declared field, calls a user-supplied
`resolveApiKey(key, req)` hook, and:
- Returns `401` if the key is absent or the hook returns null.
- Injects the resolved identity into ctx if valid.

## Change Surface

1. **New factory `api-key-route.ts`**: `apiKeyRoute(node, options)` — mutates
   the context type to add `{ apiKey: ResolvedKeyIdentity }`.
2. **`server.ts`**: If the matched route context carries an `apiKey` contract,
   extract the key from the declared location, call `resolveApiKey`, and
   short-circuit with `401` on failure.
3. **Plan 23 addendum**: API key routes emit a `securitySchemes` entry in the
   OpenAPI spec.
4. Tests: missing key → 401; invalid key (hook returns null) → 401;
   valid key → identity available in ctx.

## Dependencies

- Plan 39 (SEC2 JWT/session) — establishes the `protectedRoute` composition
  pattern; API key auth follows the same shape.
- Plan 30 (typed request headers) — `via: 'header'` variant uses the header
  schema infrastructure.

## Success Criteria

- Requests missing the declared API key return 401.
- A valid key calls `resolveApiKey`; the result is typed in ctx.
- `npm test && npm run typecheck` clean.
