# PLAN.09 — Encode path segments in `buildUrl`

## Problem

`buildUrl` in `walk.ts` interpolates path parameters via `String(allParams[seg.name])`
without calling `encodeURIComponent`. Query parameters are encoded, but path
segments containing `/`, `%`, spaces, or unicode characters will produce broken
URLs.

## Change

1. Wrap path-segment interpolation in `encodeURIComponent`:

   ```typescript
   seg.kind === 'lit' ? seg.value : encodeURIComponent(String(allParams[seg.name]));
   ```

2. Add tests in `walk.test.ts` for params with spaces, slashes, and unicode.
3. Verify roundtrip: `parse(print(route))` still works when the param value
   needs encoding (the parse side splits on `/` after the browser has already
   decoded, so this may need `decodeURIComponent` on the parse side too).

## Risk

Low, but verify the roundtrip — encoding on print without decoding on parse
would break symmetry.
